# Scalability — Coding Questions

> All implementations are in **C#**. Consistent hashing is fully implementable in C# using `List<T>.BinarySearch()` for O(log n) ring lookups, `MD5` for stable hashing, and `Dictionary` for the ring map.

## Q1. Implement a Basic Consistent Hash Ring

> Given a list of servers, implement a consistent hash ring where you can:
> - Add a server
> - Remove a server
> - Find which server a given key maps to

### Key C# Tools
- `List<uint>.BinarySearch()` — O(log n) position lookup (equivalent to Python's `bisect`)
- `System.Security.Cryptography.MD5` — stable hash across runs (unlike `GetHashCode()` which is not stable)
- Bitwise complement `~idx` — converts BinarySearch "not found" result to insertion point

```csharp
using System;
using System.Collections.Generic;
using System.Security.Cryptography;
using System.Text;

public class ConsistentHashRing
{
    private readonly Dictionary<uint, string> _ring = new();
    private readonly List<uint> _sortedKeys = new();   // always kept sorted

    private uint Hash(string key)
    {
        using var md5 = MD5.Create();
        byte[] bytes = md5.ComputeHash(Encoding.UTF8.GetBytes(key));
        return BitConverter.ToUInt32(bytes, 0);
    }

    public void AddServer(string server)
    {
        uint position = Hash(server);
        _ring[position] = server;

        int idx = _sortedKeys.BinarySearch(position);
        if (idx < 0)
            _sortedKeys.Insert(~idx, position);  // ~idx = correct insertion point
    }

    public void RemoveServer(string server)
    {
        uint position = Hash(server);
        _ring.Remove(position);

        int idx = _sortedKeys.BinarySearch(position);
        if (idx >= 0)
            _sortedKeys.RemoveAt(idx);
    }

    public string? GetServer(string key)
    {
        if (_ring.Count == 0) return null;

        uint position = Hash(key);
        int idx = _sortedKeys.BinarySearch(position);

        if (idx < 0)
            idx = ~idx;  // first position clockwise greater than hash(key)

        if (idx >= _sortedKeys.Count)
            idx = 0;     // wrap around — past last server, go back to first

        return _ring[_sortedKeys[idx]];
    }
}

// Usage
var ring = new ConsistentHashRing();
ring.AddServer("server_A");
ring.AddServer("server_B");
ring.AddServer("server_C");

Console.WriteLine(ring.GetServer("user:1"));   // → one of the servers
Console.WriteLine(ring.GetServer("user:2"));   // → consistent result every time

ring.RemoveServer("server_B");
Console.WriteLine(ring.GetServer("user:2"));   // → remapped to A or C
```

### How BinarySearch works here
```
_sortedKeys = [100, 400, 700]
Hash("user:1") = 250

BinarySearch(250) → not found → returns ~1 (idx = 1)
→ first server clockwise is at position 400 → server_B

Hash("user:2") = 800
BinarySearch(800) → not found → returns ~3 (idx = 3)
→ idx >= count → wrap around → idx = 0 → position 100 → server_A
```

**Complexity:** AddServer O(log n), RemoveServer O(log n), GetServer O(log n)

## Q2. Implement with Virtual Nodes

> Extend the ring to support virtual nodes per server. Each physical server gets `virtualNodes` positions on the ring for better load distribution.

```csharp
public class ConsistentHashRingWithVNodes
{
    private readonly Dictionary<uint, string> _ring = new();
    private readonly List<uint> _sortedKeys = new();
    private readonly int _virtualNodes;

    public ConsistentHashRingWithVNodes(int virtualNodes = 100)
    {
        _virtualNodes = virtualNodes;
    }

    private uint Hash(string key)
    {
        using var md5 = MD5.Create();
        byte[] bytes = md5.ComputeHash(Encoding.UTF8.GetBytes(key));
        return BitConverter.ToUInt32(bytes, 0);
    }

    public void AddServer(string server)
    {
        for (int i = 0; i < _virtualNodes; i++)
        {
            // Each virtual node gets unique key: "server_A#0", "server_A#1", ...
            uint position = Hash($"{server}#{i}");
            _ring[position] = server;  // maps virtual node back to real server

            int idx = _sortedKeys.BinarySearch(position);
            if (idx < 0)
                _sortedKeys.Insert(~idx, position);
        }
    }

    public void RemoveServer(string server)
    {
        for (int i = 0; i < _virtualNodes; i++)
        {
            uint position = Hash($"{server}#{i}");
            _ring.Remove(position);

            int idx = _sortedKeys.BinarySearch(position);
            if (idx >= 0)
                _sortedKeys.RemoveAt(idx);
        }
    }

    public string? GetServer(string key)
    {
        if (_ring.Count == 0) return null;

        uint position = Hash(key);
        int idx = _sortedKeys.BinarySearch(position);

        if (idx < 0) idx = ~idx;
        if (idx >= _sortedKeys.Count) idx = 0;

        return _ring[_sortedKeys[idx]];
    }

    // Helper: show distribution of keys across servers
    public Dictionary<string, int> GetDistribution(List<string> keys)
    {
        var distribution = new Dictionary<string, int>();
        foreach (var key in keys)
        {
            string? server = GetServer(key);
            if (server != null)
                distribution[server] = distribution.GetValueOrDefault(server) + 1;
        }
        return distribution;
    }
}

// Usage
var ring = new ConsistentHashRingWithVNodes(virtualNodes: 100);
ring.AddServer("server_A");
ring.AddServer("server_B");
ring.AddServer("server_C");

var keys = Enumerable.Range(0, 10000).Select(i => $"key:{i}").ToList();
var dist = ring.GetDistribution(keys);

foreach (var (server, count) in dist)
    Console.WriteLine($"{server}: {count} keys ({count / 100.0:F1}%)");

// Output (approximate):
// server_A: 3312 keys (33.1%)
// server_B: 3367 keys (33.7%)
// server_C: 3321 keys (33.2%)
```

### Why Virtual Nodes Help
```
Without virtual nodes (3 servers):
Ring: [server_A at 100] [server_B at 400] [server_C at 700]
server_A handles keys 701–100  (40% of ring)
server_B handles keys 101–400  (30% of ring)
server_C handles keys 401–700  (30% of ring)
→ Uneven distribution

With 100 virtual nodes (3 servers × 100 positions = 300 points):
Each server gets ~33% of the ring
→ Even distribution regardless of where servers land
```

## Q3. Count Keys Redistributed After Adding a Server

> Given a set of keys already distributed across N servers, how many keys need to move when you add a new server? The answer should approach **1/N** of total keys.

```csharp
public static int CountKeysRedistributed(
    List<string> existingServers,
    string newServer,
    List<string> keys)
{
    // Distribution BEFORE adding new server
    var ringBefore = new ConsistentHashRing();
    foreach (var s in existingServers)
        ringBefore.AddServer(s);

    // Distribution AFTER adding new server
    var ringAfter = new ConsistentHashRing();
    foreach (var s in existingServers)
        ringAfter.AddServer(s);
    ringAfter.AddServer(newServer);

    int moved = 0;
    foreach (var key in keys)
    {
        if (ringBefore.GetServer(key) != ringAfter.GetServer(key))
            moved++;
    }

    return moved;
}

// Usage
var servers = new List<string> { "A", "B", "C" };
var keys = Enumerable.Range(0, 10000).Select(i => $"key:{i}").ToList();

int moved = CountKeysRedistributed(servers, "D", keys);
Console.WriteLine($"Keys moved: {moved} / {keys.Count}");
Console.WriteLine($"Percentage: {moved * 100.0 / keys.Count:F1}%");

// Expected output:
// Keys moved: ~2500 / 10000
// Percentage: ~25.0%   (1/N = 1/4 when adding 4th server)
```

### What to Say in the Interview
```
Adding 1 server to N existing servers:
→ Only keys that mapped to the adjacent server need to move
→ That's approximately 1/N of all keys
→ With modulo hashing: ALL keys would need to be remapped
→ Consistent hashing: only ~25% when going from 3 to 4 servers
```

## Q4. Find K Replica Servers for a Key

> For replication (like Cassandra's replication factor), find the next K **unique** physical servers clockwise from the key's position. Virtual nodes of the same server should not count as separate replicas.

```csharp
public List<string> GetReplicaServers(string key, int k)
{
    if (_ring.Count == 0) return new List<string>();

    uint position = Hash(key);
    int startIdx = _sortedKeys.BinarySearch(position);
    if (startIdx < 0) startIdx = ~startIdx;
    if (startIdx >= _sortedKeys.Count) startIdx = 0;

    var replicas = new List<string>();
    var seenServers = new HashSet<string>();
    int i = 0;

    while (replicas.Count < k && i < _sortedKeys.Count)
    {
        int idx = (startIdx + i) % _sortedKeys.Count;
        string server = _ring[_sortedKeys[idx]];

        // Skip virtual nodes of already-selected servers
        if (!seenServers.Contains(server))
        {
            replicas.Add(server);
            seenServers.Add(server);
        }
        i++;
    }

    return replicas;
}

// Usage — replication factor 2
var ring = new ConsistentHashRingWithVNodes(100);
ring.AddServer("node_1");
ring.AddServer("node_2");
ring.AddServer("node_3");

var replicas = ring.GetReplicaServers("user:alice", k: 2);
Console.WriteLine(string.Join(", ", replicas));
// → "node_2, node_3"  (primary + 1 replica)
```

## Q5. Weighted Consistent Hashing

> Servers have different capacities. A server with weight 3 should receive approximately 3x more keys than a server with weight 1.

```csharp
public class WeightedConsistentHashRing
{
    private readonly Dictionary<uint, string> _ring = new();
    private readonly List<uint> _sortedKeys = new();
    private readonly int _baseVirtualNodes;

    public WeightedConsistentHashRing(int baseVirtualNodes = 100)
    {
        _baseVirtualNodes = baseVirtualNodes;
    }

    private uint Hash(string key)
    {
        using var md5 = MD5.Create();
        byte[] bytes = md5.ComputeHash(Encoding.UTF8.GetBytes(key));
        return BitConverter.ToUInt32(bytes, 0);
    }

    // Weight controls number of virtual nodes → controls share of ring
    public void AddServer(string server, int weight = 1)
    {
        int vnodes = _baseVirtualNodes * weight;
        for (int i = 0; i < vnodes; i++)
        {
            uint position = Hash($"{server}#{i}");
            _ring[position] = server;

            int idx = _sortedKeys.BinarySearch(position);
            if (idx < 0)
                _sortedKeys.Insert(~idx, position);
        }
    }

    public string? GetServer(string key)
    {
        if (_ring.Count == 0) return null;

        uint position = Hash(key);
        int idx = _sortedKeys.BinarySearch(position);

        if (idx < 0) idx = ~idx;
        if (idx >= _sortedKeys.Count) idx = 0;

        return _ring[_sortedKeys[idx]];
    }
}

// Usage
var ring = new WeightedConsistentHashRing(baseVirtualNodes: 100);
ring.AddServer("small_server",  weight: 1);  // 100 virtual nodes → ~25% of keys
ring.AddServer("medium_server", weight: 2);  // 200 virtual nodes → ~50% of keys
ring.AddServer("large_server",  weight: 1);  // 100 virtual nodes → ~25% of keys

var keys = Enumerable.Range(0, 10000).Select(i => $"key:{i}").ToList();
var dist = new Dictionary<string, int>();

foreach (var key in keys)
{
    string? server = ring.GetServer(key);
    if (server != null)
        dist[server] = dist.GetValueOrDefault(server) + 1;
}

foreach (var (server, count) in dist)
    Console.WriteLine($"{server}: {count} keys ({count / 100.0:F1}%)");

// Output (approximate):
// small_server:  2489 keys (24.9%)
// medium_server: 5021 keys (50.2%)
// large_server:  2490 keys (24.9%)
```

## Complexity Summary

| Operation | Time | Space |
|-----------|------|-------|
| `AddServer` | O(V log N) | O(V × S) |
| `RemoveServer` | O(V log N) | — |
| `GetServer` | O(log N) | — |
| `GetReplicas(k)` | O(k log N) | O(k) |

V = virtual nodes per server, N = total ring positions, S = number of servers

## What Interviewers Test

```
1. Do you use BinarySearch for O(log n) lookup?
   → Linear scan O(n) is a red flag

2. Do you handle ring wrap-around?
   → Forgetting idx >= count → idx = 0 is the most common bug

3. Do you use a stable hash function?
   → GetHashCode() in C# is NOT stable across runs
   → Must use MD5, SHA1, or MurmurHash

4. Can you explain WHY virtual nodes help?
   → Even distribution, not just how to implement them

5. Can you estimate keys redistributed?
   → ~1/N keys move when adding 1 server to N servers
   → Modulo hashing would move almost ALL keys

6. In GetReplicas — do you skip virtual nodes of the same server?
   → 3 replicas must be 3 DIFFERENT physical servers
```
