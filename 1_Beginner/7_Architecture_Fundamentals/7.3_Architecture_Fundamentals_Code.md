## Q1. Implement a Fixed Window Rate Limiter (and demonstrate the boundary burst bug)

> Count requests per fixed time window per client. Reject if over the limit. Then demonstrate the double-limit boundary burst problem that makes this algorithm weaker than sliding window.

### The Boundary Burst Problem
```
Limit: 5 requests per minute

t=00:59  → requests 1-5 allowed (window 1 fills up)
t=01:00  → NEW window — counter resets to 0
t=01:01  → requests 6-10 allowed (window 2 starts fresh)

Result: 10 requests in 2 seconds — 2x the intended limit
Fixed window allows bursting at window boundaries.
```

```csharp
using System;
using System.Collections.Generic;

public class FixedWindowRateLimiter
{
    private class WindowEntry
    {
        public int Count        { get; set; }
        public DateTime WindowStart { get; init; }
    }

    private readonly int _limit;
    private readonly TimeSpan _window;
    private readonly Dictionary<string, WindowEntry> _windows = new();
    private readonly object _lock = new();

    public FixedWindowRateLimiter(int requestLimit, TimeSpan window)
    {
        _limit  = requestLimit;
        _window = window;
    }

    public bool Allow(string clientId)
    {
        lock (_lock)
        {
            DateTime now = DateTime.UtcNow;

            if (!_windows.TryGetValue(clientId, out var entry) ||
                now - entry.WindowStart >= _window)
            {
                // Start a new window
                _windows[clientId] = new WindowEntry { Count = 1, WindowStart = now };
                return true;
            }

            if (entry.Count < _limit)
            {
                entry.Count++;
                return true;
            }

            return false; // limit exceeded
        }
    }

    public (int Count, TimeSpan Remaining) GetStatus(string clientId)
    {
        lock (_lock)
        {
            if (!_windows.TryGetValue(clientId, out var entry))
                return (0, _window);

            DateTime now = DateTime.UtcNow;
            if (now - entry.WindowStart >= _window)
                return (0, _window);

            return (entry.Count, _window - (now - entry.WindowStart));
        }
    }
}

// Usage — demonstrate boundary burst
var limiter = new FixedWindowRateLimiter(requestLimit: 5, window: TimeSpan.FromSeconds(60));

Console.WriteLine("=== Normal usage: 5 requests in window ===");
for (int i = 1; i <= 6; i++)
{
    bool allowed = limiter.Allow("user:alice");
    Console.WriteLine($"  Request {i}: {(allowed ? "ALLOWED" : "DENIED")}");
}

Console.WriteLine("\n=== Boundary burst demo (conceptual) ===");
Console.WriteLine(@"
  t=00:58  Request 1  → ALLOWED  (window 1, count=1)
  t=00:59  Request 2  → ALLOWED  (window 1, count=2)
  t=00:59  Request 3  → ALLOWED  (window 1, count=3)
  t=00:59  Request 4  → ALLOWED  (window 1, count=4)
  t=00:59  Request 5  → ALLOWED  (window 1, count=5)  ← window 1 full

  t=01:00  NEW WINDOW — counter resets

  t=01:00  Request 6  → ALLOWED  (window 2, count=1)
  t=01:00  Request 7  → ALLOWED  (window 2, count=2)
  t=01:00  Request 8  → ALLOWED  (window 2, count=3)
  t=01:00  Request 9  → ALLOWED  (window 2, count=4)
  t=01:00  Request 10 → ALLOWED  (window 2, count=5)

  ⚠ 10 requests in 2 seconds — 2× the intended limit of 5/min
  This is the boundary burst bug. Use Sliding Window to fix it.
");

// HTTP response headers that should accompany a 429
var (count, remaining) = limiter.GetStatus("user:alice");
Console.WriteLine("HTTP 429 response headers:");
Console.WriteLine($"  X-RateLimit-Limit:     5");
Console.WriteLine($"  X-RateLimit-Remaining: {Math.Max(0, 5 - count)}");
Console.WriteLine($"  Retry-After:           {(int)remaining.TotalSeconds}s");
```

### When Fixed Window is Still Acceptable
```
Use Fixed Window when:
  → Simplicity matters more than accuracy
  → Limits are generous (boundary burst is a small % of total)
  → You're protecting against sustained abuse, not edge bursts
  → Used by: basic API gateways, simple login throttling

Use Sliding Window when:
  → Accuracy matters (financial APIs, per-user billing)
  → Boundary burst would cause real harm
  → Memory cost is acceptable
```

**Complexity:** Allow O(1), Space O(clients)

## Q2. Implement a Leaky Bucket Rate Limiter

> Requests enter a queue (the bucket). A worker drains the queue at a fixed rate. If the bucket is full, new requests are rejected. This produces a smooth, constant output rate regardless of bursty input.

### Leaky Bucket vs Token Bucket
```
Token Bucket:  clients send at any rate up to capacity, server absorbs bursts
               → Output rate is bursty (mirrors input up to capacity)

Leaky Bucket:  requests queue up, server processes at fixed rate
               → Output rate is perfectly smooth regardless of input

Use Leaky Bucket when protecting a downstream service from any burst at all.
Use Token Bucket when you want to allow controlled bursting.
```

```csharp
public class LeakyBucketRateLimiter
{
    private readonly int _bucketCapacity; // max queued requests
    private readonly double _leakRatePerSecond; // requests processed per second

    private readonly Dictionary<string, (Queue<DateTime> Queue, DateTime LastLeak)> _buckets = new();
    private readonly object _lock = new();

    public LeakyBucketRateLimiter(int bucketCapacity, double leakRatePerSecond)
    {
        _bucketCapacity    = bucketCapacity;
        _leakRatePerSecond = leakRatePerSecond;
    }

    public bool Allow(string clientId)
    {
        lock (_lock)
        {
            DateTime now = DateTime.UtcNow;

            if (!_buckets.TryGetValue(clientId, out var bucket))
            {
                bucket = (new Queue<DateTime>(), now);
                _buckets[clientId] = bucket;
            }

            // Drain the bucket — remove requests that have been "processed"
            double elapsed   = (now - bucket.LastLeak).TotalSeconds;
            int    toRemove  = (int)(elapsed * _leakRatePerSecond);

            for (int i = 0; i < toRemove && bucket.Queue.Count > 0; i++)
                bucket.Queue.Dequeue();

            if (toRemove > 0)
                _buckets[clientId] = (bucket.Queue, now);

            // Try to add request to bucket
            if (bucket.Queue.Count < _bucketCapacity)
            {
                bucket.Queue.Enqueue(now);
                return true;
            }

            return false; // bucket full — reject
        }
    }

    public int QueueDepth(string clientId)
    {
        lock (_lock)
        {
            return _buckets.TryGetValue(clientId, out var b) ? b.Queue.Count : 0;
        }
    }
}

// Usage — downstream DB can only handle 3 queries/second
var limiter = new LeakyBucketRateLimiter(bucketCapacity: 5, leakRatePerSecond: 3);

Console.WriteLine("=== Burst of 8 requests arrives instantly ===");
for (int i = 1; i <= 8; i++)
{
    bool allowed = limiter.Allow("service:order");
    Console.WriteLine($"  Request {i:00}: {(allowed ? "ALLOWED" : "REJECTED")}  " +
                      $"(queue depth: {limiter.QueueDepth("service:order")})");
}

Console.WriteLine("\n=== After 1 second (3 processed, 2 remain) ===");
System.Threading.Thread.Sleep(1000);
Console.WriteLine($"  Queue depth after 1s: {limiter.QueueDepth("service:order")}");
bool next = limiter.Allow("service:order");
Console.WriteLine($"  New request: {(next ? "ALLOWED" : "REJECTED")}");

// Output:
// === Burst of 8 requests arrives instantly ===
//   Request 01: ALLOWED   (queue depth: 1)
//   Request 02: ALLOWED   (queue depth: 2)
//   Request 03: ALLOWED   (queue depth: 3)
//   Request 04: ALLOWED   (queue depth: 4)
//   Request 05: ALLOWED   (queue depth: 5)
//   Request 06: REJECTED  (bucket full)
//   Request 07: REJECTED  (bucket full)
//   Request 08: REJECTED  (bucket full)
//
// === After 1 second (3 processed, 2 remain) ===
//   Queue depth after 1s: 2
//   New request: ALLOWED
```

### Real-World Usage
```
Leaky Bucket protects a service with a fixed processing rate:
  → Database connection pool (max 100 concurrent queries)
  → Third-party API with hard rate limit (Stripe: 100 req/s)
  → Video transcoding pipeline (3 concurrent jobs max)
  → Print queue, task queue, any fixed-capacity processor

In practice, a real message queue (RabbitMQ, SQS) IS a leaky bucket:
  → Producer publishes at any rate
  → Consumer processes at its own fixed rate
  → Queue buffers the difference
  → If queue fills up → producer gets backpressure
```

**Complexity:** Allow O(leak count) ≈ O(1) amortized, Space O(clients × capacity)

## Q3. Implement JWT-Based Stateless Authentication

> Encode a user's identity and claims into a signed token. The server validates the signature on every request — no database lookup, no session store. Any server can validate any token.

### Why JWT Makes Services Stateless
```
Stateful (session-based):
  Login → Server stores { user_id:1, role:"admin" } in Redis
  Every request → server looks up session in Redis
  Problem: all servers must share Redis; Redis is now a SPOF

Stateless (JWT):
  Login → Server signs { user_id:1, role:"admin" } → returns token
  Every request → server validates signature, reads claims from token
  No database, no Redis, no shared state — any server can verify
```

```csharp
using System.Text;
using System.Security.Cryptography;
using System.Text.Json;

public class JwtService
{
    private readonly string _secret;

    public JwtService(string secret)
    {
        _secret = secret;
    }

    // Encode: build header.payload.signature
    public string Encode(Dictionary<string, object> claims, TimeSpan expiry)
    {
        // Add standard claims
        claims["iat"] = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        claims["exp"] = DateTimeOffset.UtcNow.Add(expiry).ToUnixTimeSeconds();

        string header  = Base64UrlEncode(JsonSerializer.Serialize(
            new { alg = "HS256", typ = "JWT" }));
        string payload = Base64UrlEncode(JsonSerializer.Serialize(claims));

        string signature = Sign($"{header}.{payload}");
        return $"{header}.{payload}.{signature}";
    }

    // Decode and verify: returns claims if valid, throws if tampered or expired
    public Dictionary<string, JsonElement> Decode(string token)
    {
        string[] parts = token.Split('.');
        if (parts.Length != 3)
            throw new ArgumentException("Invalid token format");

        // 1. Verify signature — prevents tampering
        string expectedSig = Sign($"{parts[0]}.{parts[1]}");
        if (!CryptographicOperations.FixedTimeEquals(
            Encoding.UTF8.GetBytes(expectedSig),
            Encoding.UTF8.GetBytes(parts[2])))
        {
            throw new UnauthorizedAccessException("Invalid signature — token tampered");
        }

        // 2. Decode payload
        string json   = Base64UrlDecode(parts[1]);
        var claims    = JsonSerializer.Deserialize<Dictionary<string, JsonElement>>(json)!;

        // 3. Check expiry
        long exp = claims["exp"].GetInt64();
        if (DateTimeOffset.UtcNow.ToUnixTimeSeconds() > exp)
            throw new UnauthorizedAccessException("Token expired");

        return claims;
    }

    private string Sign(string data)
    {
        using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(_secret));
        byte[] hash    = hmac.ComputeHash(Encoding.UTF8.GetBytes(data));
        return Base64UrlEncode(hash);
    }

    private static string Base64UrlEncode(string input) =>
        Base64UrlEncode(Encoding.UTF8.GetBytes(input));

    private static string Base64UrlEncode(byte[] input) =>
        Convert.ToBase64String(input)
            .TrimEnd('=')
            .Replace('+', '-')
            .Replace('/', '_');

    private static string Base64UrlDecode(string input)
    {
        string padded = input.Replace('-', '+').Replace('_', '/');
        switch (padded.Length % 4)
        {
            case 2: padded += "=="; break;
            case 3: padded += "=";  break;
        }
        return Encoding.UTF8.GetString(Convert.FromBase64String(padded));
    }
}

// Usage
var jwt = new JwtService(secret: "super-secret-key-min-32-chars!!");

Console.WriteLine("=== User logs in ===");
var claims = new Dictionary<string, object>
{
    ["user_id"] = 42,
    ["email"]   = "alice@example.com",
    ["role"]    = "admin"
};
string token = jwt.Encode(claims, expiry: TimeSpan.FromHours(1));
Console.WriteLine($"Token: {token[..50]}...  (truncated)");

Console.WriteLine("\n=== Request arrives with token ===");
var decoded = jwt.Decode(token);
Console.WriteLine($"  user_id: {decoded["user_id"]}");
Console.WriteLine($"  email:   {decoded["email"]}");
Console.WriteLine($"  role:    {decoded["role"]}");
Console.WriteLine("  → Any server can verify this — no Redis, no DB lookup");

Console.WriteLine("\n=== Tampered token is rejected ===");
string tampered = token[..^5] + "XXXXX"; // corrupt the signature
try   { jwt.Decode(tampered); }
catch (UnauthorizedAccessException ex)
{ Console.WriteLine($"  Rejected: {ex.Message}"); }

Console.WriteLine("\n=== Expired token is rejected ===");
string expired = jwt.Encode(
    new Dictionary<string, object> { ["user_id"] = 1 },
    expiry: TimeSpan.FromMilliseconds(1));
System.Threading.Thread.Sleep(10);
try   { jwt.Decode(expired); }
catch (UnauthorizedAccessException ex)
{ Console.WriteLine($"  Rejected: {ex.Message}"); }

// Output:
// Token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX...
// user_id: 42 | email: alice@example.com | role: admin
// Rejected: Invalid signature — token tampered
// Rejected: Token expired
```

### JWT Trade-offs
```
Advantages (stateless):
  ✓ No server-side session store needed
  ✓ Any server validates any token — horizontal scaling is trivial
  ✓ Works across domains (mobile app, browser, microservices)

Disadvantages:
  ✗ Cannot invalidate a token before expiry
    → User changes password → old tokens still valid until exp
    → Fix: short expiry (15min) + refresh token pattern
    → Or: maintain a token denylist in Redis (partially stateful again)

  ✗ Token grows with claims — sent on every request (bandwidth)

  ✗ Secret key must be shared across all servers securely
    → Use asymmetric keys (RS256): private key signs, public key verifies
    → Services only need the public key — private key stays with auth server
```

**Complexity:** Encode O(claims), Decode O(claims) — HMAC is O(token length)

## Q4. Implement a Service Registry with Health Checks

> Services register themselves on startup. Callers look up healthy instances by service name. A health monitor periodically checks each service and removes dead ones — eliminating the SPOF of manual service management.

```csharp
public class ServiceRegistry
{
    private class ServiceInstance
    {
        public string ServiceName  { get; init; } = "";
        public string Host         { get; init; } = "";
        public int Port            { get; init; }
        public bool IsHealthy      { get; set; } = true;
        public DateTime LastSeen   { get; set; } = DateTime.UtcNow;
        public int FailCount       { get; set; } = 0;

        public string Address => $"{Host}:{Port}";
    }

    private readonly Dictionary<string, List<ServiceInstance>> _registry = new();
    private readonly object _lock = new();
    private readonly int _failThreshold;
    private readonly TimeSpan _ttl; // deregister if not seen within TTL

    private int _roundRobinIndex = 0;

    public ServiceRegistry(int failThreshold = 3, int ttlSeconds = 30)
    {
        _failThreshold = failThreshold;
        _ttl           = TimeSpan.FromSeconds(ttlSeconds);
    }

    // Services call this on startup and periodically (heartbeat)
    public void Register(string serviceName, string host, int port)
    {
        lock (_lock)
        {
            if (!_registry.ContainsKey(serviceName))
                _registry[serviceName] = new List<ServiceInstance>();

            var existing = _registry[serviceName]
                .FirstOrDefault(s => s.Host == host && s.Port == port);

            if (existing != null)
            {
                existing.LastSeen  = DateTime.UtcNow;
                existing.IsHealthy = true;
                existing.FailCount = 0;
            }
            else
            {
                _registry[serviceName].Add(new ServiceInstance
                {
                    ServiceName = serviceName, Host = host, Port = port
                });
                Console.WriteLine($"[REGISTRY] Registered: {serviceName} @ {host}:{port}");
            }
        }
    }

    public void Deregister(string serviceName, string host, int port)
    {
        lock (_lock)
        {
            if (_registry.TryGetValue(serviceName, out var instances))
            {
                int removed = instances.RemoveAll(s => s.Host == host && s.Port == port);
                if (removed > 0)
                    Console.WriteLine($"[REGISTRY] Deregistered: {serviceName} @ {host}:{port}");
            }
        }
    }

    // Returns a healthy instance (round-robin load balancing)
    public string? Discover(string serviceName)
    {
        lock (_lock)
        {
            if (!_registry.TryGetValue(serviceName, out var instances)) return null;

            var healthy = instances.Where(s => s.IsHealthy).ToList();
            if (healthy.Count == 0) return null;

            var instance = healthy[_roundRobinIndex % healthy.Count];
            _roundRobinIndex++;
            return instance.Address;
        }
    }

    // Simulate health check probe — called by a background monitor
    public void HealthCheck(string serviceName, string host, int port, bool isReachable)
    {
        lock (_lock)
        {
            if (!_registry.TryGetValue(serviceName, out var instances)) return;
            var instance = instances.FirstOrDefault(s => s.Host == host && s.Port == port);
            if (instance == null) return;

            if (isReachable)
            {
                instance.LastSeen  = DateTime.UtcNow;
                instance.FailCount = 0;
                if (!instance.IsHealthy)
                {
                    instance.IsHealthy = true;
                    Console.WriteLine($"[HEALTH] RECOVERED: {serviceName} @ {instance.Address}");
                }
            }
            else
            {
                instance.FailCount++;
                Console.WriteLine($"[HEALTH] FAIL #{instance.FailCount}: {serviceName} @ {instance.Address}");
                if (instance.FailCount >= _failThreshold)
                {
                    instance.IsHealthy = false;
                    Console.WriteLine($"[HEALTH] MARKED DOWN: {serviceName} @ {instance.Address}");
                }
            }
        }
    }

    public void PrintRegistry()
    {
        lock (_lock)
        {
            Console.WriteLine("\n[REGISTRY] Current state:");
            foreach (var (name, instances) in _registry)
            {
                foreach (var inst in instances)
                    Console.WriteLine($"  {name,-20} {inst.Address,-20} " +
                                      $"{(inst.IsHealthy ? "HEALTHY" : "DOWN   ")}  " +
                                      $"fails={inst.FailCount}");
            }
        }
    }
}

// Usage — 3 payment service instances, one goes down
var registry = new ServiceRegistry(failThreshold: 3);

// Services register on startup
registry.Register("payment-service", "10.0.0.1", 8080);
registry.Register("payment-service", "10.0.0.2", 8080);
registry.Register("payment-service", "10.0.0.3", 8080);
registry.Register("order-service",   "10.0.1.1", 9090);

Console.WriteLine("\n=== Service discovery (round-robin) ===");
for (int i = 0; i < 4; i++)
    Console.WriteLine($"  Discover payment-service → {registry.Discover("payment-service")}");

Console.WriteLine("\n=== Node 10.0.0.2 starts failing health checks ===");
registry.HealthCheck("payment-service", "10.0.0.2", 8080, isReachable: false);
registry.HealthCheck("payment-service", "10.0.0.2", 8080, isReachable: false);
registry.HealthCheck("payment-service", "10.0.0.2", 8080, isReachable: false); // marked DOWN

registry.PrintRegistry();

Console.WriteLine("\n=== Discovery now skips downed node ===");
for (int i = 0; i < 4; i++)
    Console.WriteLine($"  Discover payment-service → {registry.Discover("payment-service")}");
// Only returns 10.0.0.1 and 10.0.0.3

Console.WriteLine("\n=== Node 10.0.0.2 recovers ===");
registry.HealthCheck("payment-service", "10.0.0.2", 8080, isReachable: true);
Console.WriteLine($"  Discover → {registry.Discover("payment-service")}"); // 10.0.0.2 back

// Output:
// [REGISTRY] Registered: payment-service @ 10.0.0.1:8080
// [REGISTRY] Registered: payment-service @ 10.0.0.2:8080
// [REGISTRY] Registered: payment-service @ 10.0.0.3:8080
// [REGISTRY] Registered: order-service @ 10.0.1.1:9090
//
// === Service discovery (round-robin) ===
//   Discover payment-service → 10.0.0.1:8080
//   Discover payment-service → 10.0.0.2:8080
//   Discover payment-service → 10.0.0.3:8080
//   Discover payment-service → 10.0.0.1:8080
//
// [HEALTH] FAIL #1: payment-service @ 10.0.0.2:8080
// [HEALTH] FAIL #2: payment-service @ 10.0.0.2:8080
// [HEALTH] FAIL #3: payment-service @ 10.0.0.2:8080
// [HEALTH] MARKED DOWN: payment-service @ 10.0.0.2:8080
//
// === Discovery now skips downed node ===
//   Discover payment-service → 10.0.0.1:8080
//   Discover payment-service → 10.0.0.3:8080
//   Discover payment-service → 10.0.0.1:8080
//   Discover payment-service → 10.0.0.3:8080
```

### How Real Service Registries Work
```
Tools: Consul, Eureka (Netflix), etcd, AWS Cloud Map, Kubernetes DNS

Two discovery patterns:

Client-side discovery (this implementation):
  Client → Registry.Discover() → gets address → calls service directly
  Used by: Netflix Eureka + Ribbon, Consul

Server-side discovery:
  Client → Load Balancer → Load Balancer queries registry → routes to instance
  Used by: Kubernetes (kube-proxy + DNS), AWS ALB

Self-registration vs third-party registration:
  Self: service calls Register() on startup, sends heartbeat
  Third-party: orchestrator (Kubernetes) registers pods automatically
```

**Complexity:** Register O(n), Discover O(healthy instances), HealthCheck O(n) — n = instances per service

## Complexity Summary

| Algorithm | Allow / Lookup | Space | Key Property |
|-----------|---------------|-------|--------------|
| Fixed Window | O(1) | O(clients) | Simple but boundary burst bug |
| Leaky Bucket | O(1) amortized | O(clients × capacity) | Smooth output, protects downstream |
| JWT Encode/Decode | O(claims) | O(1) server-side | Zero shared state, scales freely |
| Service Registry | O(instances) | O(services × instances) | Auto-removes dead nodes |

## What Interviewers Test

```
1. Fixed Window — can you name the boundary burst bug?
   → 2x limit possible at window boundaries
   → Fix: Sliding Window Counter (covered in Networking section)
   → Still used in practice because it's simple and good enough

2. Leaky Bucket vs Token Bucket — what is the key difference?
   → Leaky: fixed output rate, excess rejected/queued
   → Token: allows burst up to capacity, then throttles
   → Leaky protects downstream; Token rewards clients for being bursty

3. JWT — why can't you invalidate a JWT before expiry?
   → Tokens are stateless — server has no list of issued tokens
   → Fix: short TTL (15min) + refresh tokens
   → Or: token denylist in Redis (trade statelessness for revocability)

4. JWT — what is the difference between HS256 and RS256?
   → HS256: symmetric — same secret signs and verifies (all servers need secret)
   → RS256: asymmetric — private key signs, public key verifies
   → RS256 is better for microservices: only auth server holds private key

5. Service Registry — how do you handle the registry itself as a SPOF?
   → Registry must be highly available: run 3+ instances with consensus (Raft)
   → Consul and etcd use Raft for distributed consensus
   → Kubernetes etcd is a 3-node or 5-node cluster for exactly this reason

6. Service Registry — what if a service dies without deregistering?
   → TTL-based deregistration: if no heartbeat within TTL → remove
   → Health checks: registry actively probes services (this implementation)
   → Both are used together in production
```
