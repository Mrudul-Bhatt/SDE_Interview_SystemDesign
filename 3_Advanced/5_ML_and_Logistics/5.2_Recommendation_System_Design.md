## 2. Collaborative Filtering — User vs Item Based

```mermaid
flowchart TB
    subgraph UB["User-Based CF"]
        direction LR
        UB1["Target user Alice\nrated: [A=5, B=4, C=3]"]
        UB2["Similar users:\n- Bob (sim=0.9): [A=5, B=4, D=5]\n- Carol (sim=0.7): [B=3, D=4, E=5]"]
        UB3["Unrated by Alice:\nD: 0.9×5 + 0.7×4 / (0.9+0.7) = 4.59\nE: 0.7×5 / 0.7 = 5.0"]
        UB4["Recommend E, then D"]
        UB1 --> UB2 --> UB3 --> UB4
    end

    subgraph IB["Item-Based CF (preferred)"]
        direction LR
        IB1["Alice liked: [A=5, B=4]"]
        IB2["Item similarities (pre-computed):\nsim(A, D) = 0.85\nsim(A, E) = 0.60\nsim(B, D) = 0.75\nsim(B, F) = 0.80"]
        IB3["Score D = (0.85×5 + 0.75×4) / (0.85+0.75) = 4.59\nScore F = (0.80×4) / 0.80 = 4.0\nScore E = (0.60×5) / 0.60 = 5.0"]
        IB4["Recommend E, D, F"]
        IB1 --> IB2 --> IB3 --> IB4
    end

    note["Item-based preferred:\nitem similarity is STABLE (recompute weekly)\nuser similarity changes with every new rating"]
    style note fill:#fff9c4
```

## 3. Matrix Factorization

```mermaid
flowchart TB
    subgraph MF["Matrix Factorization: R ≈ U × Vᵀ"]
        direction LR

        subgraph R["Sparse Rating Matrix R\n(n_users × n_items)"]
            RM["     A    B    C    D
Alice  5    4    ?    ?
Bob    5    ?    3    5
Carol  ?    3    ?    4
Dave   ?    ?    5    3"]
        end

        subgraph Decompose["Decompose into dense matrices"]
            UM["User Matrix U\n(n_users × k)\nAlice  [0.9, 0.1]\nBob    [0.8, 0.3]\nCarol  [0.2, 0.9]\nDave   [0.1, 0.8]"]
            VM["Item Matrix Vᵀ\n(k × n_items)\n        A     B     C     D\ndim1  0.9   0.8   0.1   0.7\ndim2  0.1   0.2   0.9   0.4"]
        end

        subgraph Predict["Predict missing entries"]
            PRED["r̂(Alice, C) = U[Alice] · V[C]\n= 0.9×0.1 + 0.1×0.9 = 0.18 (low)\nr̂(Alice, D) = 0.9×0.7 + 0.1×0.4 = 0.67 (high)\n→ Recommend D to Alice"]
        end
    end

    R --> Decompose --> Predict

    subgraph SGD["SGD Update Rule"]
        SGD1["error = r(u,i) - U[u]·V[i]\nU[u] += η × (error×V[i] - λ×U[u])\nV[i] += η × (error×U[u] - λ×V[i])\n\nη = learning rate (0.01)\nλ = regularization (0.02)"]
    end

    Decompose -. "trained via" .-> SGD
```

## 4. Two-Tower Retrieval Architecture

```mermaid
flowchart LR
    subgraph UserTower["User Tower (online)"]
        UID["user_id\n→ embedding"]
        UCTX["Context features:\ntime_of_day, device\nlocation, session_items"]
        UHID["Hidden layers\n(2×256)"]
        UVEC["User Vector\n(d=256)"]
        UID & UCTX --> UHID --> UVEC
    end

    subgraph ItemTower["Item Tower (offline, pre-computed)"]
        IID["item_id\n→ embedding"]
        IFEAT["Item features:\ntitle TF-IDF\ncategory, tags\npopularity"]
        IHID["Hidden layers\n(2×256)"]
        IVEC["Item Vector\n(d=256)"]
        IID & IFEAT --> IHID --> IVEC
    end

    subgraph Index["FAISS / HNSW Index\n(all item vectors pre-indexed)"]
        ANN["ANN Search\nquery: user_vector\nresult: top-500 nearest items\n< 10ms over 100M items"]
    end

    UVEC -->|"serve time"| ANN
    IVEC -->|"nightly batch"| Index

    subgraph Training["Training (contrastive)"]
        POS["Positive: (user, item_they_clicked)"]
        NEG["Negatives: random unclicked items\n(in-batch negative sampling)"]
        LOSS["Loss: maximize cosine(user, pos)\nminimize cosine(user, neg)"]
        POS & NEG --> LOSS
    end
```

## 5. Retrieval + Ranking Pipeline

```mermaid
sequenceDiagram
    participant U   as User (request)
    participant RS  as Retrieval Service
    participant RNK as Ranking Service
    participant PP  as Post-Processing
    participant FS  as Feature Store (Redis)
    participant IDX as Embedding Index (FAISS)

    U->>+RS: GET /recommendations {user_id, context}

    par ANN search
        RS->>IDX: user_vector → top-200 nearest items
        IDX-->>RS: [item_ids with similarity scores]
    and CF lookup
        RS->>FS: recent interactions of user
        FS-->>RS: item-based CF candidates (top-150)
    and trending
        RS->>FS: global trending list (top-100)
        FS-->>RS: trending items
    end

    RS->>RS: Merge + deduplicate → ~500 candidates
    RS->>-RNK: Forward 500 candidates

    RNK->>+FS: Fetch features (item freshness, CTR, user affinity)
    FS-->>-RNK: Feature vectors

    RNK->>RNK: Score each candidate:\nrelevance + freshness + engagement\n- position_bias_correction
    RNK->>-PP: Top-50 scored candidates

    PP->>PP: Diversity filter (max 2 per category)
    PP->>PP: Inject 1 exploration item (ε-greedy)
    PP->>PP: Apply business rules (promotions, blocks)

    PP->>U: Top-10 personalized recommendations
```

## 8. Diversity — Maximal Marginal Relevance (MMR)

```mermaid
flowchart TB
    subgraph MMR["MMR Selection Algorithm"]
        direction TB
        POOL["Candidate pool (50 items)\neach with relevance score + embedding"]

        STEP1["Step 1: Select highest-relevance item\n→ Add to output list S"]
        STEP2["Step 2: For each remaining item c:\nMMR(c) = λ × relevance(c)\n         - (1-λ) × max_{s∈S} similarity(c, s)"]
        STEP3["Step 3: Select item with highest MMR\n→ Add to S"]
        STEP4["Repeat Steps 2-3 until |S| = K"]

        POOL --> STEP1 --> STEP2 --> STEP3 --> STEP4
    end

    subgraph Example["Example (λ=0.5)"]
        direction LR
        E1["Candidate A: rel=0.9, sim_to_selected=0.8\nMMR = 0.5×0.9 - 0.5×0.8 = 0.05"]
        E2["Candidate B: rel=0.8, sim_to_selected=0.2\nMMR = 0.5×0.8 - 0.5×0.2 = 0.30  ← selected"]
        E3["B is less relevant but more diverse → preferred\nλ controls relevance-diversity trade-off"]
    end

    MMR --> Example

    note2["λ=1.0 → pure relevance ranking (no diversity)\nλ=0.0 → pure diversity (no relevance)\nλ=0.5 → balanced (typical production setting)"]
    style note2 fill:#fff9c4
```
