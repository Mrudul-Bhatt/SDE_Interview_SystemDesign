## 2. Operational Transformation — Core Transform Rules

```mermaid
flowchart TB
    subgraph II["Insert vs Insert"]
        II_In["op1 = Insert(pos=5, 'X')\nop2 = Insert(pos=3, 'Z') — applied first"]
        II_Rule["pos2 (3) < pos1 (5)\n→ op1 shifts RIGHT by 1"]
        II_Out["Transformed op1 = Insert(pos=6, 'X')"]
        II_In --> II_Rule --> II_Out
    end

    subgraph ID["Insert vs Delete"]
        ID_In["op1 = Insert(pos=5, 'X')\nop2 = Delete(pos=3) — applied first"]
        ID_Rule["pos2 (3) < pos1 (5)\n→ op1 shifts LEFT by 1"]
        ID_Out["Transformed op1 = Insert(pos=4, 'X')"]
        ID_In --> ID_Rule --> ID_Out
    end

    subgraph DI["Delete vs Insert"]
        DI_In["op1 = Delete(pos=5)\nop2 = Insert(pos=3, 'Z') — applied first"]
        DI_Rule["pos2 (3) ≤ pos1 (5)\n→ op1 shifts RIGHT by 1"]
        DI_Out["Transformed op1 = Delete(pos=6)"]
        DI_In --> DI_Rule --> DI_Out
    end

    subgraph DD["Delete vs Delete"]
        DD_In1["op1 = Delete(pos=5)\nop2 = Delete(pos=3) — applied first"]
        DD_Out1["pos2 (3) < pos1 (5) → shift LEFT\nTransformed = Delete(pos=4)"]

        DD_In2["op1 = Delete(pos=5)\nop2 = Delete(pos=5) — applied first"]
        DD_Out2["Same position → already deleted\nTransformed = NoOp ✓"]
    end
```

## 3. Concurrent Edit Convergence (Alice & Bob)

```mermaid
sequenceDiagram
    participant A  as Alice (Client)
    participant S  as Collab Server (v=0)
    participant B  as Bob (Client)

    Note over A,B: Initial doc = "Hello" (version 0)

    A->>A: type 'X' at pos 5 → local doc = "HelloX"
    B->>B: type 'Z' at pos 2 → local doc = "HeZllo"

    A->>+S: op=Insert(pos=5,'X') at v=0
    B->>+S: op=Insert(pos=2,'Z') at v=0

    Note over S: Server receives Alice first (v=0→1)
    S->>S: Apply Insert(pos=5,'X') → "HelloX" (v=1)
    S-->>-A: ACK v=1 (no transform needed, Alice was first)

    Note over S: Server receives Bob at v=0
    S->>S: Transform Bob's op against Alice's op:\nInsert(pos=2,'Z') vs Insert(pos=5,'X')\nBob's pos=2 < Alice's pos=5 → no shift\nTransformed = Insert(pos=2,'Z')
    S->>S: Apply Insert(pos=2,'Z') → "HeZlloX" (v=2)
    S-->>-B: ACK v=2 (apply transformed op to local doc)

    S->>A: Broadcast Insert(pos=2,'Z') at v=2
    S->>B: Broadcast Insert(pos=5,'X') transformed for Bob:\n(Bob already has 'Z', so Alice's pos=5→6)\nInsert(pos=6,'X')

    A->>A: Apply Insert(pos=2,'Z') → "HeZlloX" ✓
    B->>B: Apply Insert(pos=6,'X') to "HeZllo" → "HeZlloX" ✓

    Note over A,B: Both converge to "HeZlloX" ✓
```

## 5. CRDT Alternative — Unique Character IDs

```mermaid

        E["id=(s,2)\n'e'"]
        L1["id=(s,3)\n'l'"]
        L2["id=(s,4)\n'l'"]
        O["id=(s,5)\n'o'"]
        H --> E --> L1 --> L2 --> O
    end

    subgraph AliceInsert["Alice: Insert 'X' after id=(s,5)"]
        AX["New node:\nid=(alice,1)\n'X'\nafter: (s,5)"]
    end

    subgraph BobInsert["Bob: Insert 'Z' after id=(s,2) (same time)"]
        BZ["New node:\nid=(bob,1)\n'Z'\nafter: (s,2)"]
    end

    subgraph Merged["After Merge (no server transform needed)"]
        direction LR
        MH["'H'"] --> ME["'e'"] --> MZ["'Z' (bob,1)"] --> ML1["'l'"] --> ML2["'l'"] --> MO["'o'"] --> MX["'X' (alice,1)"]
    end

    subgraph Delete["Bob: Delete 'e' (tombstone)"]
        Tomb["id=(s,2): deleted=true\n(kept in list for position calc)\nrendered as invisible"]
    end

    CRDT --> AliceInsert & BobInsert --> Merged
    Merged --> Delete

    note["CRDT: no transform function needed\nConcurrent ops always merge correctly\nTrade-off: tombstones accumulate"]
    style note fill:#fff9c4
```

## 6. Client State Machine

```mermaid
flowchart TD
    subgraph ClientState["Client State per Document"]
        S1["STATE: Synchronized\nLocal = Server version\nNo pending ops"]
        S2["STATE: Pending\nLocal changes not yet ACKed\nBuffer of pending ops"]
        S3["STATE: Applying Remote\nTransforming incoming op\nagainst pending ops"]
    end

    subgraph Transitions["Transitions"]
        T1["User types\n→ Apply locally (instant)\n→ Send to server"]
        T2["Server ACKs op\n→ Remove from pending buffer\n→ If empty: back to Synchronized"]
        T3["Remote op arrives\n→ Transform against all pending ops\n→ Apply to local doc\n→ Stay in Pending"]
        T4["Reconnect after offline\n→ Send all queued ops in order"]
    end

    S1 -->|"user edit"| T1 --> S2
    S2 -->|"ACK received"| T2 --> S1
    S2 -->|"remote op received"| T3 --> S2
    S1 -->|"remote op received"| S3 --> S1
    S2 -->|"network lost"| S2
    S2 -->|"reconnect"| T4 --> S2
```

## 7. Cursor Position Synchronization

```mermaid
sequenceDiagram
    participant A  as Alice
    participant S  as Collab Server
    participant B  as Bob

    Note over A,B: Both editing doc. Alice cursor at pos=20, Bob cursor at pos=8

    A->>S: CURSOR {pos=20, color=blue}
    S->>B: CURSOR alice {pos=20, color=blue}
    Note over B: Bob sees Alice's cursor at position 20

    B->>B: Insert 'Z' at pos=5
    B->>S: op=Insert(pos=5,'Z') + CURSOR {pos=9}
    S->>A: Broadcast Insert(pos=5,'Z')

    Note over A: Remote op: Insert at pos=5 < Alice's cursor pos=20
    A->>A: Transform cursor: pos=20 → pos=21\n(one char inserted before cursor)
    Note over A: Alice's cursor auto-shifts to pos=21, stays on same character
```

## 8. Offline Sync on Reconnect

```mermaid
sequenceDiagram
    participant C  as Carol (Client)
    participant S  as Collab Server

    Note over C: Carol goes offline at v=10

    C->>C: Types 3 characters (ops 1, 2, 3) offline
    C->>C: Queued: [op1@v=10, op2@v=10, op3@v=10]

    Note over S: Server advances to v=15 with other users' edits

    Note over C: Carol reconnects
    C->>+S: Reconnect, send queued ops [op1, op2, op3] from v=10

    S->>S: Transform op1 against ops at v=11,12,13,14,15
    S->>S: Apply transformed op1 → v=16
    S->>S: Transform op2 against updated history → v=17
    S->>S: Transform op3 → v=18

    S-->>-C: ACK + send ops v=11 to v=18 (catch-up)
    C->>C: Apply catch-up ops\n(transform against local pending ops already applied)
    Note over C,S: Carol converges to v=18 ✓
```
