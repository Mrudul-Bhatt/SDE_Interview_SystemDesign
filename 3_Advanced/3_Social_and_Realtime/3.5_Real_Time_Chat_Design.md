## 2. Online Message Delivery (Alice → Bob, Both Online)

```mermaid
sequenceDiagram
    participant A  as Alice (Client)
    participant S1 as Chat Server 1
    participant DB as Cassandra
    participant Reg as Connection Registry (Redis)
    participant Bus as Message Bus (Redis)
    participant S2 as Chat Server 2
    participant B  as Bob (Client)

    A->>+S1: WS: SEND {to:bob, content:"Hey!"}

    S1->>S1: Generate message_id (Snowflake)
    S1->>+DB: INSERT message (chat_id, msg_id, sender, content, status=SENT)
    DB-->>-S1: OK

    S1->>+Reg: GET user_server:bob
    Reg-->>-S1: "Server 2"

    S1->>Bus: PUBLISH user:bob {msg_id, content, sender:alice}
    S1-->>-A: WS: ACK {msg_id, status:SENT}

    Bus->>+S2: message for bob (subscribed to user:bob)
    S2->>+B: WS: MESSAGE {msg_id, content, from:alice}
    B-->>-S2: WS: ACK DELIVERED {msg_id}

    S2->>DB: UPDATE status=DELIVERED
    S2->>Bus: PUBLISH user:alice {msg_id, status:DELIVERED}
    Bus->>S1: delivery receipt for alice
    S1->>A: WS: RECEIPT {msg_id, status:DELIVERED}

    Note over A,B: Bob opens chat
    B->>S2: WS: READ {chat_id, last_msg_id}
    S2->>DB: UPDATE status=READ
    S2->>Bus: PUBLISH user:alice {msg_id, status:READ}
    Bus->>S1: read receipt
    S1->>A: WS: RECEIPT {msg_id, status:READ} ✓✓
```

## 3. Offline Message Delivery

```mermaid
sequenceDiagram
    participant A  as Alice (Client)
    participant S1 as Chat Server 1
    participant DB as Cassandra
    participant Reg as Connection Registry (Redis)
    participant Push as Push Notification (FCM)
    participant S3  as Chat Server 3 (Bob reconnects here)
    participant B  as Bob (Device)

    Note over B: Bob is OFFLINE

    A->>+S1: WS: SEND {to:bob, content:"Where are you?"}
    S1->>DB: INSERT message (status=SENT)
    S1->>+Reg: GET user_server:bob
    Reg-->>-S1: (nil) — bob is offline
    S1->>Push: SendPush(bob_device_token, "New message from Alice")
    Push->>B: FCM push notification: "New message from Alice"
    S1-->>-A: ACK {msg_id, status:SENT}

    Note over B: Bob opens the app
    B->>+S3: WebSocket connect
    S3->>Reg: SET user_server:bob "Server3"
    S3->>+DB: Fetch undelivered messages for bob
    DB-->>-S3: [msg_id: "Where are you?"]
    S3->>B: WS: DELIVER {all pending messages}
    B-->>S3: WS: ACK DELIVERED
    S3->>DB: UPDATE status=DELIVERED (all)
    S3->>-B: (connection established)

    S3->>Bus: PUBLISH user:alice {status:DELIVERED}
    Bus->>S1: delivery receipt
    S1->>A: WS: RECEIPT ✓
```

## 4. Group Chat Fan-out

```mermaid
sequenceDiagram
    participant A  as Alice (Sender)
    participant S1 as Chat Server 1
    participant DB as Cassandra
    participant Bus as Message Bus
    participant S2 as Chat Server 2
    participant S3 as Chat Server 3
    participant B  as Bob (online, S2)
    participant C  as Carol (online, S3)
    participant D  as Dave (offline)
    participant Push as Push Service

    A->>+S1: WS: SEND {group_id:family, content:"Dinner at 7?"}
    S1->>DB: INSERT message (chat_id=family, ONE write)
    S1->>DB: GET group_members(family) → [bob, carol, dave]

    Note over S1: Fan-out to all members (async)
    S1->>Bus: PUBLISH user:bob {msg}
    S1->>Bus: PUBLISH user:carol {msg}
    S1->>Bus: PUBLISH user:dave → (nil, offline)
    S1->>Push: SendPush(dave)
    S1-->>-A: ACK SENT

    Bus->>S2: msg for bob
    S2->>B: WS: DELIVER
    B-->>S2: ACK DELIVERED

    Bus->>S3: msg for carol
    S3->>C: WS: DELIVER
    C-->>S3: ACK DELIVERED

    Note over A,Push: Group DELIVERED = all online members received
    S2->>Bus: PUBLISH user:alice {bob:DELIVERED}
    S3->>Bus: PUBLISH user:alice {carol:DELIVERED}
    Bus->>S1: receipts
    S1->>A: WS: ✓ (delivered to bob, carol)
```

## 7. Presence Service

```mermaid
sequenceDiagram
    participant C  as Client (Bob)
    participant PS as Presence Service
    participant RD as Redis

    Note over C,RD: Bob opens the app
    C->>PS: CONNECT + start heartbeat loop

    loop Every 15 seconds
        C->>+PS: HEARTBEAT {userId: bob}
        PS->>RD: SET presence:bob {ts=now} EX 30
        PS->>RD: SET last_seen:bob {ts=now}
        PS-->>-C: OK
    end

    Note over C,RD: Bob closes the app (no more heartbeats)
    Note over RD: After 30s, presence:bob key expires automatically

    Note over C,RD: Alice checks Bob's status
    C->>+PS: GET_PRESENCE {userId: bob}
    PS->>RD: EXISTS presence:bob
    RD-->>PS: 0 (expired — bob is offline)
    PS->>RD: GET last_seen:bob
    RD-->>PS: "2024-01-15 18:45:00"
    PS-->>-C: {online: false, last_seen: "2024-01-15 18:45:00"}
```

## 8. Media Sharing Flow

```mermaid
sequenceDiagram
    participant C  as Client (Alice)
    participant MS as Media Service
    participant S3 as S3 / Object Store
    participant CS as Chat Server
    participant B  as Bob (Client)

    C->>+MS: POST /media/upload-url {filename: photo.jpg, size: 150KB}
    MS->>S3: Generate presigned PUT URL (valid 5 min)
    MS-->>-C: {upload_url, media_id: "m:abc123", cdn_url}

    C->>S3: PUT photo.jpg (direct upload — bypasses chat servers)
    S3-->>C: 200 OK

    C->>+CS: WS: SEND {to:bob, type:IMAGE, media_url:"cdn.../m:abc123", content:"Check this out!"}
    CS->>CS: Persist message with media_url (tiny reference)
    CS->>B: WS: DELIVER {type:IMAGE, media_url, content}
    CS-->>-C: ACK SENT

    B->>S3: GET cdn.../m:abc123 (CDN-cached, fast)
    S3-->>B: photo.jpg (served from CDN edge)
```
