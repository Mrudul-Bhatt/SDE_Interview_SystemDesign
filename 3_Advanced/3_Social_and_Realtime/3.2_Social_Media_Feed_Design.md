## 1. High-Level System Architecture

```mermaid
flowchart TB

    subgraph Storage["Storage Layer"]
        PostDB["Post DB\n(Cassandra)\nauthor+time partitioned"]

    Mobile & Web --> API

    FanOut -->|"celebrity\nskip fan-out"| PostDB
    FeedAPI --> FeedSvc
    FeedSvc -->|"1. pre-computed feed"| FeedCache
    FeedSvc -->|"2. merge celebrity posts"| PostDB
    FollowAPI --> FollowSvc --> FollowGraph
```

## 2. Fan-out Decision: Regular User vs Celebrity

```mermaid
flowchart TD
    Post["New Post Created\nauthor: userId\ncontent: ..."]
    Store["Store in Post DB\n(Cassandra)"]
    Count["Get follower count\nfollowers:{userId}"]

    Regular{"follower_count\n< 10M?"}

    subgraph FanOutWrite["Fan-out on Write (Regular User)"]
        Followers["Load all follower IDs\n(e.g., 500 followers)"]
        Push["For each follower:\nZADD feed:{followerId} score postId"]
        Trim["Trim feed to 1000 entries\n(ZREMRANGEBYRANK)"]
        Followers --> Push --> Trim
    end

    subgraph NoFanOut["Skip Fan-out (Celebrity)"]
        Skip["Do NOT push to 50M feeds\nJust stored in Post DB"]
        Note["At read time:\nfollowers pull latest posts\nfrom this celebrity"]
        Skip --> Note
    end

    Post --> Store --> Count --> Regular
    Regular -->|"YES"| FanOutWrite
    Regular -->|"NO (celebrity)"| NoFanOut
```

## 3. Feed Read Path — Hybrid Merge

```mermaid
sequenceDiagram
    participant U  as User (Bob)
    participant FS as Feed Service
    participant RC as Redis (Feed Cache)
    participant FG as Follow Graph
    participant PD as Post DB (Cassandra)
    participant RK as Ranker

    U->>+FS: GET /feed?limit=20

    FS->>+RC: ZREVRANGE feed:bob 0 39
    RC-->>-FS: [postId_1, postId_2, ... postId_40] (pre-computed from non-celebrities)

    FS->>+FG: GetCelebrityFollows(bob)
    FG-->>-FS: [elon, mrbeast] (accounts with > 10M followers)

    FS->>+PD: GetRecentPosts(elon, limit=10)
    PD-->>-FS: [post_e1, post_e2, ...]

    FS->>+PD: GetRecentPosts(mrbeast, limit=10)
    PD-->>-FS: [post_m1, post_m2, ...]

    FS->>+PD: BatchGetPosts([postId_1...postId_40])
    PD-->>-FS: post objects with engagement counts

    FS->>+RK: Rank(all posts, userId=bob)
    Note over RK: score = engagement_decay(likes, age)
    RK-->>-FS: top 20 ranked posts

    FS-->>-U: [{post, author, likes, timestamp, ...} × 20]\nnext_cursor=timestamp_of_last
```

## 5. Cursor-Based Pagination

```mermaid
sequenceDiagram
    participant C as Client
    participant FS as Feed Service
    participant RC as Redis

    C->>FS: GET /feed?limit=20
    FS->>RC: ZREVRANGE feed:user1 0 19
    RC-->>FS: [p1(ts=1000), p2(ts=990), ... p20(ts=810)]
    FS-->>C: posts + next_cursor=810

    Note over C: User scrolls down

    C->>FS: GET /feed?before=810&limit=20
    FS->>RC: ZREVRANGEBYSCORE feed:user1 (810 -inf LIMIT 0 20
    RC-->>FS: [p21(ts=800), p22(ts=790), ... p40(ts=610)]
    FS-->>C: posts + next_cursor=610

    Note over C,RC: New posts arrive while user scrolls
    Note over C,RC: Cursor is STABLE — new posts don't shift existing ones
    Note over C,RC: vs offset: ?offset=20 would skip/duplicate after new post
```

## 6. Ranking Score Model

```mermaid
flowchart TB
    subgraph Input["Post Signals"]
        L["likes = 450"]
        CM["comments = 30"]
        SH["shares = 15"]
        A["age = 2 hours"]
        AF["author_affinity = 0.8\n(Bob interacts with Alice often)"]
    end

    subgraph Formula["Ranking Formula"]
        Raw["raw_engagement = likes×2 + comments×3 + shares×5\n= 450×2 + 30×3 + 15×5\n= 900 + 90 + 75 = 1065"]
        Decay["time_decay = 1 / (age_hours + 1)^1.8\n= 1 / (2+1)^1.8\n= 1 / 7.22 = 0.138"]
        Affinity["affinity_boost = 1 + author_affinity × 0.5\n= 1 + 0.8 × 0.5 = 1.4"]
        Score["final_score = raw_engagement × time_decay × affinity_boost\n= 1065 × 0.138 × 1.4\n= 205.8"]
    end

    subgraph Comparison["Score Comparison"]
        P1["Post A (2h old, 1065 eng): score=205.8"]
        P2["Post B (0.1h old, 50 eng): score=50/(1.1^1.8)=47.2"]
        P3["Post C (24h old, 5000 eng): score=5000/625^1.8×1=small"]
        Winner["Winner: Post A (balanced recency + engagement)"]
        P1 & P2 & P3 --> Winner
    end

    Input --> Raw & Decay & Affinity --> Score --> Comparison
```

## 7. Follow / Unfollow Flow

```mermaid
sequenceDiagram
    participant U  as User (Bob)
    participant FS as Follow Service
    participant FG as Follow Graph (Redis)
    participant FO as Fan-out Service
    participant FC as Feed Cache (Redis)
    participant PD as Post DB

    Note over U,PD: Bob follows Alice

    U->>+FS: POST /follow {target: alice}
    FS->>FG: SADD following:bob alice
    FS->>FG: SADD followers:alice bob
    FS->>+FO: Backfill(bob, alice)
    FO->>+PD: GetRecentPosts(alice, limit=20)
    PD-->>-FO: [post_a1, post_a2, ... post_a20]
    FO->>FC: ZADD feed:bob score postId (×20)
    FO-->>-FS: backfill done
    FS-->>-U: 200 OK

    Note over U,PD: Bob unfollows Alice

    U->>+FS: DELETE /follow {target: alice}
    FS->>FG: SREM following:bob alice
    FS->>FG: SREM followers:alice bob
    FS->>FO: Cleanup(bob, alice) [async background job]
    Note over FO,FC: Async: scan feed:bob, remove posts by alice
    FS-->>-U: 200 OK (feed cleanup happens in background)
```

## 8. Post Data Model (Cassandra)

```mermaid
flowchart LR
    subgraph PostTable["posts table (Cassandra)"]
        direction TB
        Schema["PRIMARY KEY: (author_id, created_at DESC, post_id)\n\nColumns:\n  author_id   TEXT\n  post_id     TEXT  (Snowflake ID)\n  content     TEXT\n  media_url   TEXT\n  like_count  COUNTER\n  created_at  TIMESTAMP\n  post_type   TEXT  (text/image/video)"]
    end

    subgraph SnowflakeID["Snowflake Post ID (64-bit)"]
        direction LR
        Bits["[41 bits: ms timestamp]\n[10 bits: machine ID]\n[12 bits: sequence]\n→ Globally unique\n→ Time-sortable\n→ No coordination needed"]
    end

    subgraph ReadPattern["Common Read Patterns"]
        R1["Get user's posts:\nSELECT * FROM posts\nWHERE author_id='alice'\nLIMIT 20\n→ Single partition scan"]
        R2["Time-range query:\nWHERE author_id='alice'\nAND created_at > '2024-01-01'\n→ Efficient with clustering key"]
    end

    PostTable --> SnowflakeID
    PostTable --> ReadPattern
```
