```mermaid

    end

        N3["Node C\nPartitions: 171–255\n(primary + replicas)"]

        N2 <-->|"Gossip\nReplication"| N3
        N3 <-->|"Gossip\nReplication"| N1
    end

    subgraph Storage["Per-Node Storage Engine"]
        WAL["WAL\n(Write-Ahead Log)"]

        BF -.->|"skip miss"| SST
    end

    C1 & C2 & C3 --> Router
    Router -->|"hash(key) → node"| N1 & N2 & N3
    N1 --> Storage
```

## 3. Write Path — LSM Tree

```mermaid
sequenceDiagram
    participant C  as Client
    participant Co as Coordinator
    participant P  as Primary Node
    participant R1 as Replica 1
    participant R2 as Replica 2
    participant D  as Disk (SSTable)

    C->>+Co: PUT("user:42", "{name:Alice}", TTL=3600)
    Co->>Co: hash("user:42") → Node B (primary)
    Co->>+P: forward PUT

    P->>P: 1. Append to WAL (fsync)
    P->>P: 2. Write to MemTable (sorted)
    P->>+R1: async replicate
    P->>+R2: async replicate
    R1-->>-P: ACK
    P-->>-Co: ACK (W=2 quorum satisfied)
    Co-->>-C: OK

    Note over P: MemTable reaches 64 MB threshold
    P->>+D: Flush MemTable → SSTable L0
    D-->>-P: L0 file written

    Note over P,D: Background compaction
    P->>D: Merge L0 files → L1 (sort + deduplicate)
    D-->>P: L1 compacted
```

## 4. Read Path — Quorum Read with Read Repair

```mermaid
sequenceDiagram
    participant C  as Client
    participant Co as Coordinator
    participant N1 as Node 1 (primary)
    participant N2 as Node 2 (replica)
    participant N3 as Node 3 (replica — stale)

    C->>+Co: GET("user:42") [R=2]
    Co->>N1: read request
    Co->>N2: read request
    Co->>N3: read request

    N1-->>Co: value=v3 (ts=1000)
    N2-->>Co: value=v3 (ts=1000)
    N3-->>Co: value=v2 (ts=900) ← STALE

    Co->>Co: Quorum of 2 agree on v3 (ts=1000)
    Co-->>-C: return v3

    Note over Co,N3: Read Repair (async)
    Co->>N3: PUT("user:42", v3, ts=1000)
    N3-->>Co: ACK (now up-to-date)
```

## 5. Replication — Quorum vs Consistency Trade-off

```mermaid
flowchart TB
    subgraph Config["RF=3 Configuration Options"]
        direction LR
        W1R1["W=1 / R=1\nFastest\nEventual consistency\n(stale reads possible)"]
        W2R2["W=2 / R=2\nBalanced\nMost stale reads caught\n(W+R=4 > RF=3)"]
        W3R3["W=3 / R=3\nSlowest\nLinearizable reads\n(every node must respond)"]
    end

    subgraph CAP["CAP Position"]
        AP["AP Mode\nW=1 / R=1\nAlways available\nFinal consistency"]
        CP["CP Mode\nW=3 / R=3\nConsistent\nMay reject during partition"]
    end

    W1R1 --> AP
    W3R3 --> CP
    W2R2 -->|"sweet spot"| Note1

    Note1["Use case match:\nSession store → W=1/R=1\nShopping cart → W=2/R=2\nBank balance → W=3/R=3\nor use single-node ACID DB"]
    style Note1 fill:#e8f5e9
```

## 6. Node Storage Engine — LSM Layers

```mermaid
flowchart TB
    subgraph WriteP["Write Path"]
        W["PUT(key, value)"]
        WAL["WAL\n(append-only log, fsync)\nCrash recovery"]
        MT["MemTable\n(Red-Black tree)\n64 MB limit"]
        W --> WAL --> MT
    end

    subgraph ReadP["Read Path"]
        R["GET(key)"]
        MT2["MemTable\n(check first — freshest)"]
        BF["Bloom Filter\n(99% skip rate for missing keys)"]
        L0["L0 SSTables\n(newest flush, unsorted across files)"]
        L1["L1 SSTables\n(sorted, 10× L0 size)"]
        L2["L2 SSTables\n(sorted, 10× L1 size)"]
        MISS["Key not found\n→ return null"]
        R --> MT2
        MT2 -->|"not in MemTable"| BF
        BF -->|"definitely absent"| MISS
        BF -->|"maybe present"| L0 --> L1 --> L2 --> MISS
    end

    subgraph Compact["Background Compaction"]
        C["Merge L0 → L1\nRemove tombstones\nApply TTL expiry\nSort keys"]
    end

    MT -->|"flush on size limit"| L0
    L0 & L1 -->|"trigger"| C
```

## 7. Failure Detection — Gossip Protocol

```mermaid
sequenceDiagram
    participant A as Node A
    participant B as Node B
    participant C as Node C
    participant D as Node D (failing)

    Note over A,D: Normal operation — every 3s
    A->>B: gossip({A:alive@t=100, D:alive@t=97})
    B->>C: gossip({A:alive@t=100, B:alive@t=100, D:alive@t=97})
    C->>A: gossip({B:alive@t=100, C:alive@t=100, D:alive@t=97})

    Note over D: Node D crashes (t=100)

    A->>B: gossip({A:alive@t=103, D:alive@t=97})
    Note over A,B: D not heard from for 9s (3×T) → suspect
    B->>C: gossip({D:suspect@t=109})
    C->>A: gossip({D:suspect@t=109})

    Note over A,B,C: After 2 more rounds without D → mark DOWN
    A->>A: Reassign D's partitions\n(hinted handoff activated)
```

## 8. Hinted Handoff — Write During Node Failure

```mermaid
flowchart TB
    subgraph Normal["Normal Write: RF=3"]
        W["PUT(key=X)"]
        N1["Node 1 (primary) ✓"]
        N2["Node 2 (replica) ✓"]
        N3["Node 3 (replica) ✓"]
        W --> N1 & N2 & N3
    end

    subgraph Failure["Node 3 is DOWN"]
        W2["PUT(key=X)"]
        N1B["Node 1 (primary) ✓"]
        N2B["Node 2 (replica) ✓"]
        N4["Node 4 (hint holder)\nStores write with metadata:\n'deliver to Node 3 when back'"]
        W2 --> N1B & N2B & N4
    end

    subgraph Recovery["Node 3 Recovers"]
        N3C["Node 3 rejoins cluster"]
        N4C["Node 4 detects N3 is alive"]
        Deliver["Node 4 delivers buffered hints to Node 3"]
        N3C --> N4C --> Deliver
    end

    Normal --> Failure --> Recovery
```
