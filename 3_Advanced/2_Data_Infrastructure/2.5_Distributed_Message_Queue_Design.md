## 3. Write Path — Producer to Broker

```mermaid
sequenceDiagram
    participant Prod as Producer
    participant Meta as Metadata Cache
    participant Lead as Partition Leader (Broker 1)
    participant Rep1 as Replica (Broker 2) — ISR
    participant Rep2 as Replica (Broker 3) — ISR

    Prod->>Meta: Which broker is leader for orders/P0?
    Meta-->>Prod: Broker 1

    Note over Prod: Batch messages (linger.ms=5ms, batch.size=16KB)
    Prod->>+Lead: PRODUCE [msg1, msg2, msg3] acks=-1

    Lead->>Lead: Append to partition log (sequential write)
    Lead->>+Rep1: Replicate [msg1, msg2, msg3]
    Lead->>+Rep2: Replicate [msg1, msg2, msg3]
    Rep1-->>-Lead: ACK (ISR member)
    Rep2-->>-Lead: ACK (ISR member)

    Lead-->>-Prod: ACK offset=102 (all ISR confirmed)

    Note over Lead: acks=1 variant: ACK after local write only
    Note over Lead: acks=0 variant: no ACK at all (fire-and-forget)
```

## 4. Consumer Group — Partition Assignment and Rebalance

```mermaid
sequenceDiagram
    participant C1 as Consumer 1
    participant C2 as Consumer 2
    participant GC as Group Coordinator (Broker)
    participant C3 as Consumer 3 (joining)

    Note over C1,C2: Initial state: 3 partitions, 2 consumers
    C1->>GC: JoinGroup(group=billing)
    C2->>GC: JoinGroup(group=billing)
    GC-->>C1: Assignment: [P0, P1]
    GC-->>C2: Assignment: [P2]

    C1->>GC: Heartbeat (every 3s)
    C2->>GC: Heartbeat (every 3s)

    Note over C3: New consumer joins
    C3->>+GC: JoinGroup(group=billing)
    GC->>C1: Revoke all partitions (rebalance)
    GC->>C2: Revoke all partitions (rebalance)
    C1-->>GC: Partitions revoked
    C2-->>GC: Partitions revoked

    GC-->>C1: New assignment: [P0]
    GC-->>C2: New assignment: [P1]
    GC-->>-C3: New assignment: [P2]

    Note over C1,C3: 1 partition per consumer — maximum parallelism
```

## 5. Delivery Guarantees Comparison

```mermaid
flowchart TB
    subgraph AtMostOnce["At-Most-Once"]
        direction LR
        AMO1["1. Consumer polls message"]
        AMO2["2. Commit offset"]
        AMO3["3. Process message"]
        AMO4["CRASH HERE → message LOST\n(offset already committed)"]
        AMO1 --> AMO2 --> AMO3
        AMO2 -.->|"crash after commit"| AMO4
    end

    subgraph AtLeastOnce["At-Least-Once (default)"]
        direction LR
        ALO1["1. Consumer polls message"]
        ALO2["2. Process message"]
        ALO3["3. Commit offset"]
        ALO4["CRASH AFTER PROCESS,\nBEFORE COMMIT →\nmessage re-processed\n(idempotent consumer needed)"]
        ALO1 --> ALO2 --> ALO3
        ALO2 -.->|"crash after process"| ALO4
    end

    subgraph ExactlyOnce["Exactly-Once (Kafka Transactions)"]
        direction LR
        EOT1["1. Begin transaction"]
        EOT2["2. Process message"]
        EOT3["3. Atomic: write output\n+ commit offset\nin one transaction"]
        EOT4["CRASH → transaction\nrolled back → retry\nfrom same offset"]
        EOT1 --> EOT2 --> EOT3
        EOT2 -.->|"crash = rollback"| EOT4
    end
```

## 6. ISR Replication and Leader Election

```mermaid
flowchart TB
    subgraph Normal["Normal Operation"]
        Lead["Leader: Broker 1\noffset=100"]
        ISR1["Broker 2 (ISR)\noffset=99"]
        ISR2["Broker 3 (ISR)\noffset=100"]
        NonISR["Broker 4 (NOT ISR)\noffset=85 — too far behind"]
        Lead -->|"replicate"| ISR1 & ISR2
        Lead -.->|"replicate (slow)"| NonISR
    end

    subgraph Failure["Broker 1 Crashes"]
        ZK["ZooKeeper / KRaft\ndetects leader down"]
        Elect["Elect new leader\nfrom ISR only\n{Broker2, Broker3}\n→ pick Broker 3 (most caught up)"]
        NewLead["New Leader: Broker 3\noffset=100"]
        ZK --> Elect --> NewLead
    end

    subgraph Risk["Why NOT elect non-ISR?"]
        Risk1["Broker 4 at offset=85\nElecting it = lose offsets 86–100\n= 15 messages LOST\n→ Forbidden (unclean.leader.election=false)"]
    end

    Normal --> Failure
    Failure --> Risk
```

## 8. Consumer Lag Monitoring

```mermaid
flowchart TB
    subgraph Metrics["Key Metrics"]
        M1["latest_offset\n(partition head)"]
        M2["committed_offset\n(consumer position)"]
        M3["consumer_lag =\nlatest - committed"]
    end

    subgraph States["Consumer Lag States"]
        S0["lag = 0\nConsumer caught up ✓"]
        S1["lag = 1K–100K\nNormal processing backlog"]
        S2["lag > 100K\nAlert: consumer falling behind"]
        S3["lag > retention\nDANGER: offsets expire!\nMessages will be lost"]
    end

    subgraph Actions["Remediation"]
        A1["Scale: add consumers\n(up to partition count)"]
        A2["Optimize: reduce\nprocessing time per message"]
        A3["Increase: retention.ms\nto buy more time"]
    end

    M3 --> S0 & S1 & S2 & S3
    S2 --> A1 & A2
    S3 --> A3
```
