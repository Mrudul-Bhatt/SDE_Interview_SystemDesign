## 2. Chunked Resumable Upload

```mermaid
sequenceDiagram
    participant C  as Client
    participant US as Upload Service
    participant S3 as Raw Video Store (S3)
    participant KF as Kafka

    C->>+US: POST /upload/init {filename, size, mime}
    US-->>-C: {upload_id, chunk_urls[0..N]}

    loop For each 5MB chunk
        C->>+US: PUT /upload/{id}/chunk/{n} [bytes + MD5]
        US->>S3: Write chunk to temp path
        US-->>-C: 200 OK (chunk received)
    end

    C->>+US: POST /upload/{id}/complete
    US->>S3: Assemble chunks → raw/{video_id}/original.mp4
    US->>KF: Publish "upload-complete" {video_id, uploader_id, size}
    US-->>-C: 202 Accepted (transcoding queued)

    Note over C,KF: Client can resume from any chunk on network failure\nPOST /upload/{id}/status → returns last received chunk
```

```mermaid

    end

    SEG --> R360 & R720 & R1080

    THUMB --> THUMBS
```

## 4. HLS Adaptive Bitrate Playback

```mermaid
sequenceDiagram
    participant P  as ABR Player (client)
    participant CDN as CDN Edge
    participant ORI as S3 Origin

    P->>+CDN: GET hls/{vid}/manifest.m3u8
    CDN->>ORI: Cache MISS → fetch manifest
    CDN-->>-P: Master playlist (lists 360p/480p/720p/1080p)

    Note over P: Player selects 720p (default)
    P->>+CDN: GET hls/{vid}/720p/index.m3u8
    CDN-->>-P: 720p segment list

    loop Every 6-second segment
        P->>+CDN: GET hls/{vid}/720p/seg042.ts
        CDN-->>-P: Segment bytes
        P->>P: Measure download speed\n(throughput = bytes / elapsed)
        alt speed > 5 Mbps AND buffer > 20s
            P->>P: Switch to 1080p next segment
        else speed < 1 Mbps OR buffer < 5s
            P->>P: Drop to 480p next segment
        end
    end

    Note over P,ORI: Segments are immutable → CDN TTL = ∞\nHIT rate 95%+ for popular videos
```

```mermaid
flowchart TB

        V["Browser / App\nABR Player"]
    end

    end

    subgraph CDN_Region["CDN Regional Origin Cache"]
        REG_CACHE["Regional Cache\n(Top 30% videos)\nFallback for edge miss"]
    end

    subgraph S3_Origin["S3 CDN Origin (AWS)"]
        S3["All transcoded segments\n(permanent storage)"]
    end

    V -->|"segment request"| EDGE_CACHE
    EDGE_CACHE -->|"HIT (95%+ popular)"| V
    EDGE_CACHE -->|"MISS"| REG_CACHE
    REG_CACHE -->|"HIT"| EDGE_CACHE
    REG_CACHE -->|"MISS"| S3
    S3 -->|"fill"| REG_CACHE

    note_pop["DNS geo-routes viewer to nearest PoP\nSegments content-addressed → infinite TTL"]
    style note_pop fill:#fff9c4
```

## 8. Transcoding Queue — Priority and Failure Handling

```mermaid
flowchart TB
    subgraph Queues["Kafka Topics (by video length)"]
        Q_SHORT["short-videos\n< 1 minute\n(TikTok-style)\nHigh priority"]
        Q_MED["medium-videos\n1–20 minutes\n(YouTube default)"]
        Q_LONG["long-videos\n> 20 minutes\n(movies, lectures)\nLow priority"]
    end

    subgraph Workers["Worker Pools"]
        W_SHORT["4× small GPU workers\n(fast turnaround)"]
        W_MED["8× medium GPU workers"]
        W_LONG["4× large GPU workers\n(high memory)"]
    end

    subgraph Failure["Failure Handling"]
        RETRY["Retry queue\n(3 attempts, exp backoff)"]
        DLQ["Dead Letter Queue\n(alert oncall after 3 fails)"]
        IDEM["Idempotent: re-transcode\nwrites to new segment path;\noriginal always in S3"]
    end

    Q_SHORT --> W_SHORT
    Q_MED --> W_MED
    Q_LONG --> W_LONG

    W_SHORT & W_MED & W_LONG -->|"failure"| RETRY
    RETRY -->|"3rd failure"| DLQ
    RETRY -->|"retry OK"| Failure
    IDEM -. "design principle" .- W_SHORT & W_MED & W_LONG

    note["Priority queues prevent long 4K uploads\nblocking short viral clips"]
    style note fill:#fff9c4
```
