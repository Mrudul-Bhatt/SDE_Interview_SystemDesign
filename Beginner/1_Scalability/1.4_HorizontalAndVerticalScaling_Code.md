# Horizontal & Vertical Scaling — Coding Questions

> Vertical scaling has no direct coding questions — it's a hardware/config decision. Horizontal scaling directly leads to **load balancing algorithm** questions, since distributing traffic across multiple servers requires smart routing logic.

---

## Q1. Implement Round Robin Load Balancer

> Given a list of servers, distribute requests evenly across them in order. Handle servers being added and removed dynamically.

```csharp
using System;
using System.Collections.Generic;

public class RoundRobinLoadBalancer
{
    private readonly List<string> _servers = new();
    private int _currentIndex = 0;
    private readonly object _lock = new(); // thread-safe for concurrent requests

    public void AddServer(string server) => _servers.Add(server);

    public void RemoveServer(string server)
    {
        int idx = _servers.IndexOf(server);
        if (idx == -1) return;
        _servers.RemoveAt(idx);
        // Adjust index if removed server was at or before current position
        if (idx <= _currentIndex)
            _currentIndex = Math.Max(0, _currentIndex - 1);
    }

    public string? GetServer()
    {
        if (_servers.Count == 0) return null;
        lock (_lock)
        {
            string server = _servers[_currentIndex];
            _currentIndex = (_currentIndex + 1) % _servers.Count;
            return server;
        }
    }
}

// Usage
var lb = new RoundRobinLoadBalancer();
lb.AddServer("S1");
lb.AddServer("S2");
lb.AddServer("S3");

for (int i = 0; i < 6; i++)
    Console.WriteLine(lb.GetServer());
// S1, S2, S3, S1, S2, S3
```

### How the Wrap-Around Works
```
_servers = [S1, S2, S3]
_currentIndex cycles: 0 → 1 → 2 → 0 → 1 → 2 → ...

Request 1: returns S1, index becomes 1
Request 2: returns S2, index becomes 2
Request 3: returns S3, index becomes 0  ← modulo wrap
Request 4: returns S1, index becomes 1
```

**Complexity:** GetServer O(1), AddServer O(1), RemoveServer O(n)

## Q2. Implement Weighted Round Robin

> Servers have different capacities. A server with weight 3 should handle 3x more requests than a server with weight 1.

```csharp
public class WeightedRoundRobinLoadBalancer
{
    private readonly List<(string Server, int Weight)> _servers = new();
    private List<string> _sequence = new(); // expanded weighted sequence
    private int _currentIndex = 0;

    public void AddServer(string server, int weight)
    {
        _servers.Add((server, weight));
        RebuildSequence();
    }

    public void RemoveServer(string server)
    {
        _servers.RemoveAll(s => s.Server == server);
        RebuildSequence();
    }

    private void RebuildSequence()
    {
        _sequence = new List<string>();
        foreach (var (server, weight) in _servers)
            for (int i = 0; i < weight; i++)
                _sequence.Add(server);
        _currentIndex = 0;
    }

    public string? GetServer()
    {
        if (_sequence.Count == 0) return null;
        string server = _sequence[_currentIndex];
        _currentIndex = (_currentIndex + 1) % _sequence.Count;
        return server;
    }
}

// Usage
var lb = new WeightedRoundRobinLoadBalancer();
lb.AddServer("S1", weight: 1);
lb.AddServer("S2", weight: 2);
lb.AddServer("S3", weight: 1);

for (int i = 0; i < 8; i++)
    Console.Write(lb.GetServer() + " ");
// S1 S2 S2 S3 S1 S2 S2 S3
```

### How the Sequence Is Built
```
Weights: S1=1, S2=2, S3=1
Expanded sequence: [S1, S2, S2, S3]  (total = sum of weights = 4)

Requests cycle through this sequence:
Request 1 → S1
Request 2 → S2
Request 3 → S2  ← S2 gets 2 out of every 4 requests
Request 4 → S3
Request 5 → S1  (wraps around)
```

**Complexity:** GetServer O(1), RebuildSequence O(total weight)

## Q3. Implement Least Connections Load Balancer

> Route each request to the server with the fewest active connections. When a request completes, the connection count decreases.

```csharp
public class LeastConnectionsLoadBalancer
{
    private readonly Dictionary<string, int> _connections = new();
    private readonly object _lock = new();

    public void AddServer(string server)
    {
        lock (_lock) { _connections[server] = 0; }
    }

    public void RemoveServer(string server)
    {
        lock (_lock) { _connections.Remove(server); }
    }

    public string? GetServer()
    {
        if (_connections.Count == 0) return null;
        lock (_lock)
        {
            string? best = null;
            int minConn = int.MaxValue;

            foreach (var (server, count) in _connections)
            {
                if (count < minConn)
                {
                    minConn = count;
                    best = server;
                }
            }

            if (best != null) _connections[best]++;
            return best;
        }
    }

    public void ReleaseConnection(string server)
    {
        lock (_lock)
        {
            if (_connections.ContainsKey(server) && _connections[server] > 0)
                _connections[server]--;
        }
    }

    public void PrintConnections()
    {
        foreach (var (server, count) in _connections)
            Console.WriteLine($"{server}: {count} active connections");
    }
}

// Usage
var lb = new LeastConnectionsLoadBalancer();
lb.AddServer("S1");
lb.AddServer("S2");
lb.AddServer("S3");

string? r1 = lb.GetServer(); // S1 (all at 0, picks first) → S1 now has 1
string? r2 = lb.GetServer(); // S2 (S2,S3 at 0) → S2 now has 1
string? r3 = lb.GetServer(); // S3 (S3 at 0) → S3 now has 1
string? r4 = lb.GetServer(); // all tied at 1, picks S1 → S1 now has 2

lb.ReleaseConnection("S1"); // S1 back to 1
lb.ReleaseConnection("S1"); // S1 back to 0

string? r5 = lb.GetServer(); // S1 wins (0 connections)
lb.PrintConnections();
// S1: 1, S2: 1, S3: 1
```

### When to Use Each Algorithm

```
Round Robin:
  → All servers have equal capacity
  → Requests take similar time to process
  → Stateless APIs, web servers

Weighted Round Robin:
  → Servers have different CPU/RAM (heterogeneous fleet)
  → You know the relative capacity of each server upfront

Least Connections:
  → Request duration varies significantly (some fast, some slow)
  → Long-lived connections: WebSockets, file uploads, video streaming
  → Better than Round Robin when one slow request can clog a server
```

## Q4. Load Balancer with Health Checks

> Extend a load balancer to skip unhealthy servers. Mark a server DOWN after N consecutive failures. Recover it after M consecutive successes.

```csharp
public class HealthAwareLoadBalancer
{
    private class ServerState
    {
        public string Name        { get; init; } = "";
        public bool IsHealthy     { get; set; }  = true;
        public int ConsecFailures { get; set; }  = 0;
        public int ConsecSuccesses{ get; set; }  = 0;
    }

    private readonly List<ServerState> _servers = new();
    private int _currentIndex = 0;
    private readonly int _failThreshold;
    private readonly int _recoverThreshold;

    public HealthAwareLoadBalancer(int failThreshold = 3, int recoverThreshold = 2)
    {
        _failThreshold    = failThreshold;
        _recoverThreshold = recoverThreshold;
    }

    public void AddServer(string server) =>
        _servers.Add(new ServerState { Name = server });

    public string? GetServer()
    {
        int attempts = 0;
        while (attempts < _servers.Count)
        {
            var state = _servers[_currentIndex];
            _currentIndex = (_currentIndex + 1) % _servers.Count;

            if (state.IsHealthy) return state.Name;
            attempts++;
        }
        return null; // all servers are down
    }

    public void ReportSuccess(string server)
    {
        var state = _servers.Find(s => s.Name == server);
        if (state == null) return;

        state.ConsecFailures = 0;
        if (!state.IsHealthy)
        {
            state.ConsecSuccesses++;
            if (state.ConsecSuccesses >= _recoverThreshold)
            {
                state.IsHealthy      = true;
                state.ConsecSuccesses = 0;
                Console.WriteLine($"[RECOVERED] {server} is back online");
            }
        }
    }

    public void ReportFailure(string server)
    {
        var state = _servers.Find(s => s.Name == server);
        if (state == null) return;

        state.ConsecSuccesses = 0;
        state.ConsecFailures++;
        if (state.ConsecFailures >= _failThreshold && state.IsHealthy)
        {
            state.IsHealthy = false;
            Console.WriteLine($"[DOWN] {server} marked as unhealthy");
        }
    }

    public void PrintStatus()
    {
        foreach (var s in _servers)
            Console.WriteLine($"{s.Name}: {(s.IsHealthy ? "HEALTHY" : "DOWN")}");
    }
}

// Usage
var lb = new HealthAwareLoadBalancer(failThreshold: 3, recoverThreshold: 2);
lb.AddServer("S1");
lb.AddServer("S2");
lb.AddServer("S3");

lb.ReportFailure("S2");
lb.ReportFailure("S2");
lb.ReportFailure("S2"); // [DOWN] S2 marked as unhealthy

// S2 is now skipped
Console.WriteLine(lb.GetServer()); // S1
Console.WriteLine(lb.GetServer()); // S3 (S2 skipped)
Console.WriteLine(lb.GetServer()); // S1

lb.ReportSuccess("S2");
lb.ReportSuccess("S2"); // [RECOVERED] S2 is back online

lb.PrintStatus();
// S1: HEALTHY
// S2: HEALTHY
// S3: HEALTHY
```

## Complexity Summary

| Algorithm | GetServer | AddServer | RemoveServer | Best For |
|-----------|-----------|-----------|--------------|----------|
| Round Robin | O(1) | O(1) | O(n) | Equal capacity servers |
| Weighted RR | O(1) | O(W) | O(W) | Heterogeneous capacity |
| Least Connections | O(n) | O(1) | O(1) | Variable request duration |
| Health-Aware RR | O(n) worst | O(1) | O(n) | Fault-tolerant systems |

W = total weight, n = number of servers

## What Interviewers Test

```
1. Round Robin — do you handle the wrap-around with modulo?
   → _currentIndex = (_currentIndex + 1) % _servers.Count

2. Weighted RR — do you rebuild the sequence on server changes?
   → Forgetting to reset _currentIndex after rebuild is a common bug

3. Least Connections — do you use a lock for thread safety?
   → GetServer + increment must be atomic, not two separate operations

4. Health checks — do you skip ALL unhealthy servers or just one?
   → Must loop up to _servers.Count times, not just once

5. Can you explain WHEN to use each algorithm?
   → Least Connections > Round Robin for WebSockets/file uploads
   → Weighted RR when you have a mix of t2.micro and c5.4xlarge servers

6. What happens if ALL servers go down?
   → Return null / throw exception, don't infinite loop
```
