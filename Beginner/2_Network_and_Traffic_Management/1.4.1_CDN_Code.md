# CDN — Coding Questions

> CDN edge nodes are essentially **smart caches close to the user**. Interview questions test your ability to implement the eviction and expiration policies that keep those caches efficient. All implementations are in **C#**.

---

## Q1. Implement an LRU Cache

> Design a cache with a fixed capacity. On a cache miss, fetch the value and store it. When the cache is full, evict the **least recently used** item. All operations must be O(1).

### Key C# Tools
- `Dictionary<K, LinkedListNode<V>>` — O(1) key lookup
- `LinkedList<(K key, V value)>` — O(1) move-to-front and remove from tail
- The combination gives O(1) get, put, and eviction

```csharp
using System;
using System.Collections.Generic;

public class LRUCache<TKey, TValue> where TKey : notnull
{
    private readonly int _capacity;
    private readonly Dictionary<TKey, LinkedListNode<(TKey Key, TValue Value)>> _map = new();
    private readonly LinkedList<(TKey Key, TValue Value)> _list = new();

    public LRUCache(int capacity)
    {
        _capacity = capacity;
    }

    public TValue? Get(TKey key)
    {
        if (!_map.TryGetValue(key, out var node))
            return default; // cache miss

        // Move to front (most recently used)
        _list.Remove(node);
        _list.AddFirst(node);
        return node.Value.Value;
    }

    public void Put(TKey key, TValue value)
    {
        if (_map.TryGetValue(key, out var existing))
        {
            // Update value, move to front
            _list.Remove(existing);
            _map.Remove(key);
        }
        else if (_map.Count == _capacity)
        {
            // Evict LRU (tail of list)
            var lru = _list.Last!;
            _list.RemoveLast();
            _map.Remove(lru.Value.Key);
        }

        var node = _list.AddFirst((key, value));
        _map[key] = node;
    }

    public bool ContainsKey(TKey key) => _map.ContainsKey(key);

    public void PrintCache()
    {
        Console.Write("Cache (MRU → LRU): ");
        foreach (var (k, v) in _list)
            Console.Write($"[{k}:{v}] ");
        Console.WriteLine();
    }
}

// Usage — CDN edge node with capacity 3
var cache = new LRUCache<string, string>(capacity: 3);

cache.Put("homepage.html", "<html>home</html>");
cache.Put("logo.png",      "binary_data_logo");
cache.Put("style.css",     "body { margin: 0 }");
cache.PrintCache();
// Cache (MRU → LRU): [style.css:...] [logo.png:...] [homepage.html:...]

cache.Get("homepage.html"); // accessed → moves to front
cache.PrintCache();
// Cache (MRU → LRU): [homepage.html:...] [style.css:...] [logo.png:...]

cache.Put("video.mp4", "binary_data_video"); // capacity full → evict LRU (logo.png)
cache.PrintCache();
// Cache (MRU → LRU): [video.mp4:...] [homepage.html:...] [style.css:...]

Console.WriteLine(cache.Get("logo.png")); // → null (evicted)
```

### How the Linked List + HashMap Works
```
Capacity = 3. State after Put(A), Put(B), Put(C):
  List: [C] ↔ [B] ↔ [A]   (head = MRU, tail = LRU)
  Map:  {A→nodeA, B→nodeB, C→nodeC}

Get(A):  move nodeA to head
  List: [A] ↔ [C] ↔ [B]

Put(D):  evict tail (B), add D at head
  List: [D] ↔ [A] ↔ [C]
  Map:  {A→nodeA, C→nodeC, D→nodeD}
```

**Complexity:** Get O(1), Put O(1), Space O(capacity)

## Q2. Implement an LFU Cache

> Evict the **least frequently used** item when the cache is full. On a tie in frequency, evict the **least recently used** among them.

```csharp
public class LFUCache<TKey, TValue> where TKey : notnull
{
    private readonly int _capacity;
    private int _minFreq = 0;

    private readonly Dictionary<TKey, (TValue Value, int Freq)> _keyMap = new();
    // freq → ordered set of keys (insertion order = LRU within same freq)
    private readonly Dictionary<int, LinkedList<TKey>> _freqMap = new();
    private readonly Dictionary<TKey, LinkedListNode<TKey>> _nodeMap = new();

    public LFUCache(int capacity)
    {
        _capacity = capacity;
    }

    public TValue? Get(TKey key)
    {
        if (!_keyMap.TryGetValue(key, out var entry))
            return default;

        IncrementFreq(key, entry.Freq);
        return entry.Value;
    }

    public void Put(TKey key, TValue value)
    {
        if (_capacity <= 0) return;

        if (_keyMap.TryGetValue(key, out var existing))
        {
            _keyMap[key] = (value, existing.Freq);
            IncrementFreq(key, existing.Freq);
            return;
        }

        if (_keyMap.Count == _capacity)
            EvictLFU();

        _keyMap[key] = (value, 1);
        if (!_freqMap.ContainsKey(1))
            _freqMap[1] = new LinkedList<TKey>();

        var node = _freqMap[1].AddLast(key);
        _nodeMap[key] = node;
        _minFreq = 1;
    }

    private void IncrementFreq(TKey key, int freq)
    {
        // Remove from current freq bucket
        var node = _nodeMap[key];
        _freqMap[freq].Remove(node);
        if (_freqMap[freq].Count == 0)
        {
            _freqMap.Remove(freq);
            if (_minFreq == freq) _minFreq++;
        }

        // Add to next freq bucket
        int newFreq = freq + 1;
        if (!_freqMap.ContainsKey(newFreq))
            _freqMap[newFreq] = new LinkedList<TKey>();

        var newNode = _freqMap[newFreq].AddLast(key);
        _nodeMap[key] = newNode;
        _keyMap[key] = (_keyMap[key].Value, newFreq);
    }

    private void EvictLFU()
    {
        var bucket = _freqMap[_minFreq];
        var lruKey = bucket.First!.Value; // LRU within min-freq bucket
        bucket.RemoveFirst();
        if (bucket.Count == 0) _freqMap.Remove(_minFreq);

        _keyMap.Remove(lruKey);
        _nodeMap.Remove(lruKey);
    }
}

// Usage
var cache = new LFUCache<int, string>(capacity: 3);

cache.Put(1, "homepage");   // freq: {1:1}
cache.Put(2, "logo");       // freq: {1:1, 2:1}
cache.Put(3, "style");      // freq: {1:1, 2:1, 3:1}

cache.Get(1); // freq: {1:2, 2:1, 3:1}
cache.Get(1); // freq: {1:3, 2:1, 3:1}
cache.Get(2); // freq: {1:3, 2:2, 3:1}

// Cache full — add key 4, evict LFU (key 3, freq=1)
cache.Put(4, "video");
Console.WriteLine(cache.Get(3)); // → null (evicted)
Console.WriteLine(cache.Get(1)); // → "homepage"
Console.WriteLine(cache.Get(4)); // → "video"
```

### LRU vs LFU — When to Use Each
```
LRU (Least Recently Used):
  → Good for temporal locality (recently accessed = likely accessed again soon)
  → Used in: CDN edge caches, CPU L1/L2 caches, browser caches
  → Problem: a one-time viral video can flush the entire cache

LFU (Least Frequently Used):
  → Good for frequency locality (popular content stays cached)
  → Used in: database buffer pools, long-lived CDN caches
  → Problem: old popular content can block new trending content
```

**Complexity:** Get O(1), Put O(1), Space O(capacity)

## Q3. Implement a TTL Cache

> Cache entries expire after a set time-to-live (TTL). Expired entries should not be returned and should be cleaned up. This models how CDN edge nodes respect `Cache-Control: max-age` headers.

```csharp
public class TTLCache<TKey, TValue> where TKey : notnull
{
    private class CacheEntry
    {
        public TValue Value      { get; init; } = default!;
        public DateTime ExpiresAt { get; init; }
        public bool IsExpired => DateTime.UtcNow >= ExpiresAt;
    }

    private readonly Dictionary<TKey, CacheEntry> _store = new();
    private readonly object _lock = new();

    public void Put(TKey key, TValue value, TimeSpan ttl)
    {
        lock (_lock)
        {
            _store[key] = new CacheEntry
            {
                Value     = value,
                ExpiresAt = DateTime.UtcNow + ttl
            };
        }
    }

    public TValue? Get(TKey key)
    {
        lock (_lock)
        {
            if (!_store.TryGetValue(key, out var entry))
                return default; // miss

            if (entry.IsExpired)
            {
                _store.Remove(key); // lazy eviction on access
                return default;     // treat as miss
            }

            return entry.Value;
        }
    }

    public bool ContainsKey(TKey key) => Get(key) is not null;

    // Proactive cleanup — call periodically (e.g., background timer)
    public int Cleanup()
    {
        lock (_lock)
        {
            var expired = new List<TKey>();
            foreach (var (key, entry) in _store)
                if (entry.IsExpired) expired.Add(key);

            foreach (var key in expired)
                _store.Remove(key);

            return expired.Count;
        }
    }
}

// Usage — CDN edge node respecting Cache-Control headers
var cache = new TTLCache<string, string>();

cache.Put("homepage.html", "<html>...</html>", ttl: TimeSpan.FromSeconds(2));
cache.Put("logo.png",      "binary_data",      ttl: TimeSpan.FromSeconds(10));

Console.WriteLine(cache.Get("homepage.html")); // → "<html>...</html>"
Console.WriteLine(cache.Get("logo.png"));      // → "binary_data"

System.Threading.Thread.Sleep(3000); // wait 3 seconds

Console.WriteLine(cache.Get("homepage.html")); // → null (expired after 2s)
Console.WriteLine(cache.Get("logo.png"));      // → "binary_data" (still valid)

int cleaned = cache.Cleanup();
Console.WriteLine($"Cleaned up {cleaned} expired entries");
```

### Two Eviction Strategies
```
Lazy eviction (implemented above):
  → Check expiry only on access
  → Expired keys stay in memory until accessed
  → Simple, no background threads needed

Proactive eviction (Cleanup() method):
  → Periodic scan to remove all expired keys
  → Keeps memory usage bounded
  → Used in Redis: combines both approaches
```

**Complexity:** Get O(1), Put O(1), Cleanup O(n)

## Q4. Find the Nearest CDN Edge Server

> Given a user's location (latitude, longitude) and a list of CDN edge server locations, return the closest server. This is how CDN providers route users to the optimal edge node.

```csharp
public class CDNRouter
{
    public record EdgeServer(string Name, double Lat, double Lon);

    private readonly List<EdgeServer> _servers = new();

    public void AddServer(string name, double lat, double lon)
    {
        _servers.Add(new EdgeServer(name, lat, lon));
    }

    public EdgeServer? GetNearestServer(double userLat, double userLon)
    {
        if (_servers.Count == 0) return null;

        EdgeServer? nearest = null;
        double minDistance = double.MaxValue;

        foreach (var server in _servers)
        {
            double dist = HaversineDistanceKm(userLat, userLon, server.Lat, server.Lon);
            if (dist < minDistance)
            {
                minDistance = dist;
                nearest = server;
            }
        }

        return nearest;
    }

    // Haversine formula: great-circle distance between two points on a sphere
    private static double HaversineDistanceKm(double lat1, double lon1, double lat2, double lon2)
    {
        const double R = 6371; // Earth radius in km
        double dLat = ToRad(lat2 - lat1);
        double dLon = ToRad(lon2 - lon1);

        double a = Math.Sin(dLat / 2) * Math.Sin(dLat / 2)
                 + Math.Cos(ToRad(lat1)) * Math.Cos(ToRad(lat2))
                 * Math.Sin(dLon / 2) * Math.Sin(dLon / 2);

        double c = 2 * Math.Atan2(Math.Sqrt(a), Math.Sqrt(1 - a));
        return R * c;
    }

    private static double ToRad(double deg) => deg * Math.PI / 180;

    public void PrintDistances(double userLat, double userLon)
    {
        Console.WriteLine($"User at ({userLat}, {userLon}):");
        foreach (var server in _servers)
        {
            double dist = HaversineDistanceKm(userLat, userLon, server.Lat, server.Lon);
            Console.WriteLine($"  {server.Name}: {dist:F0} km");
        }
    }
}

// Usage
var router = new CDNRouter();
router.AddServer("US-East",    lat: 39.0, lon: -77.0);   // Virginia
router.AddServer("US-West",    lat: 37.3, lon: -121.9);  // California
router.AddServer("EU-Central", lat: 50.1, lon:   8.7);   // Frankfurt
router.AddServer("AP-South",   lat: 19.0, lon:  72.8);   // Mumbai

// User in London
double userLat = 51.5, userLon = -0.1;
router.PrintDistances(userLat, userLon);
// User at (51.5, -0.1):
//   US-East:    5,580 km
//   US-West:    8,756 km
//   EU-Central:   904 km   ← nearest
//   AP-South:   7,189 km

var nearest = router.GetNearestServer(userLat, userLon);
Console.WriteLine($"\nRouted to: {nearest?.Name}");
// → Routed to: EU-Central
```

### Why Distance Alone Isn't Enough (Say This in the Interview)
```
Real CDN routing also considers:

1. Network latency  — not the same as geographic distance
   (fiber routes aren't straight lines, peering agreements matter)

2. Server load      — a closer server at 95% capacity may perform
   worse than a slightly farther one at 20% capacity

3. Anycast routing  — CDN providers use BGP anycast so the same IP
   resolves to different servers by network topology, not GPS

4. Health           — skip servers that are down or degraded

→ In practice: DNS-based routing + anycast, with latency probes
```

**Complexity:** GetNearestServer O(S) — S = number of edge servers (small, ~dozens to hundreds)

## Complexity Summary

| Algorithm | Get | Put | Eviction | Best For |
|-----------|-----|-----|----------|----------|
| LRU Cache | O(1) | O(1) | Least recently used | Temporal locality, browser/edge caches |
| LFU Cache | O(1) | O(1) | Least frequently used | Frequency locality, DB buffer pools |
| TTL Cache | O(1) | O(1) | Time-based expiry | Cache-Control headers, session tokens |
| Nearest Edge | — | — | — | Geographic CDN routing |

## What Interviewers Test

```
1. LRU — can you achieve O(1) for all operations?
   → HashMap + DoublyLinkedList is the only way
   → Array or sorted structure gives O(n) or O(log n)

2. LFU — do you handle the tie-breaking rule?
   → On tie in frequency, evict the LRU among them
   → Need per-frequency buckets (LinkedList per freq level)

3. TTL — where do you check expiry?
   → On every Get (lazy eviction)
   → Also need a periodic Cleanup() to avoid memory leaks

4. Nearest server — do you know Haversine vs Euclidean?
   → Euclidean works for small areas; Haversine for globe-scale
   → Interviewers accept either, but naming Haversine shows depth

5. Can you explain why CDN caching uses LRU?
   → Web content has strong temporal locality
   → A page viewed once is likely viewed again soon (sessions, crawlers)
   → Viral content surges are handled by TTL expiry + origin pull
```
