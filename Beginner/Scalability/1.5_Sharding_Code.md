# Sharding — Coding Questions

> Sharding questions test whether you can route keys to the correct shard, handle shard changes (resharding), and detect load imbalances (hotspots). All implementations are in **C#**.

---

## Q1. Implement a Hash-Based Shard Router

> Given N shards, route any key to the correct shard using modulo hashing. Support adding shards (with resharding cost) and looking up which shard owns a key.

```csharp
using System;
using System.Collections.Generic;

public class HashShardRouter
{
    private int _shardCount;

    public HashShardRouter(int shardCount)
    {
        _shardCount = shardCount;
    }

    public int GetShard(string key)
    {
        int hash = Math.Abs(key.GetHashCode());
        return hash % _shardCount;
    }

    // Returns how many keys need to move when shard count changes
    public int CountKeysToRemap(List<string> keys, int newShardCount)
    {
        int moved = 0;
        foreach (var key in keys)
        {
            int oldShard = Math.Abs(key.GetHashCode()) % _shardCount;
            int newShard = Math.Abs(key.GetHashCode()) % newShardCount;
            if (oldShard != newShard) moved++;
        }
        return moved;
    }

    public void Resize(int newShardCount) => _shardCount = newShardCount;
}

// Usage
var router = new HashShardRouter(shardCount: 4);

Console.WriteLine(router.GetShard("user:alice"));  // → 0, 1, 2, or 3
Console.WriteLine(router.GetShard("user:bob"));
Console.WriteLine(router.GetShard("user:alice")); // always the same shard

var keys = Enumerable.Range(0, 10000).Select(i => $"key:{i}").ToList();
int moved = router.CountKeysToRemap(keys, newShardCount: 5);
Console.WriteLine($"Keys remapped: {moved} / {keys.Count}");
// → ~8000 / 10000  (80%+ keys move — this is why modulo hashing is bad)
```

### Why Modulo Hashing is Problematic
```
4 shards: shard = hash(key) % 4
5 shards: shard = hash(key) % 5

key:1  → hash=1  → shard 1%4=1  vs  1%5=1  (stays)
key:2  → hash=2  → shard 2%4=2  vs  2%5=2  (stays)
key:5  → hash=5  → shard 5%4=1  vs  5%5=0  (MOVES)
key:6  → hash=6  → shard 6%4=2  vs  6%5=1  (MOVES)

→ ~80% of keys need to move when adding just 1 shard
→ Consistent hashing only moves ~20% (1/N of keys)
```

**Complexity:** GetShard O(1), CountKeysToRemap O(k) — k = number of keys

## Q2. Implement a Range-Based Shard Router

> Keys are strings (e.g., usernames). Route each key to a shard based on alphabetical ranges. Support dynamic range boundaries.

```csharp
public class RangeShardRouter
{
    // Each entry: (rangeEnd, shardId)
    // Keys <= rangeEnd go to that shard
    private readonly List<(string RangeEnd, int ShardId)> _ranges = new();

    // ranges: [("f", 0), ("m", 1), ("s", 2), ("\uffff", 3)]
    // means:  A-F → shard 0, G-M → shard 1, N-S → shard 2, T-Z → shard 3
    public void AddRange(string rangeEnd, int shardId)
    {
        _ranges.Add((rangeEnd, shardId));
        _ranges.Sort((a, b) => string.Compare(a.RangeEnd, b.RangeEnd, StringComparison.Ordinal));
    }

    public int GetShard(string key)
    {
        string lowerKey = key.ToLowerInvariant();

        foreach (var (rangeEnd, shardId) in _ranges)
        {
            if (string.Compare(lowerKey, rangeEnd, StringComparison.Ordinal) <= 0)
                return shardId;
        }

        // Key is beyond all ranges — goes to last shard
        return _ranges[^1].ShardId;
    }

    public void PrintRanges()
    {
        string prev = "start";
        foreach (var (rangeEnd, shardId) in _ranges)
        {
            Console.WriteLine($"Shard {shardId}: {prev} → {rangeEnd}");
            prev = rangeEnd;
        }
    }
}

// Usage
var router = new RangeShardRouter();
router.AddRange("f", shardId: 0);   // A–F → Shard 0
router.AddRange("m", shardId: 1);   // G–M → Shard 1
router.AddRange("s", shardId: 2);   // N–S → Shard 2
router.AddRange("\uffff", shardId: 3); // T–Z → Shard 3

Console.WriteLine(router.GetShard("alice"));  // → 0  (a <= f)
Console.WriteLine(router.GetShard("henry"));  // → 1  (h <= m)
Console.WriteLine(router.GetShard("oscar"));  // → 2  (o <= s)
Console.WriteLine(router.GetShard("victor")); // → 3  (v > s)

router.PrintRanges();
// Shard 0: start → f
// Shard 1: f → m
// Shard 2: m → s
// Shard 3: s → \uffff
```

### When Range-Based Sharding Causes Hotspots
```
User signups skew toward common first letters:
  Shard 0 (A-F): "adam", "alice", "anna", "bob", "charlie"... → OVERLOADED
  Shard 3 (T-Z): "tony", "zach"... → UNDERLOADED

Fix: use consistent hashing or adjust range boundaries dynamically
```

**Complexity:** GetShard O(S) — S = number of shards (typically very small)

## Q3. Resharding Calculator — Modulo vs Consistent Hashing

> Show concretely how many keys move when adding a shard with modulo hashing vs consistent hashing. The interviewer wants to see you quantify the difference.

```csharp
using System.Security.Cryptography;
using System.Text;

public static class ReshardingCalculator
{
    // Modulo hashing: shard = hash(key) % N
    public static int ModuloKeysRemapped(List<string> keys, int oldShards, int newShards)
    {
        int moved = 0;
        foreach (var key in keys)
        {
            int hash = Math.Abs(key.GetHashCode());
            if (hash % oldShards != hash % newShards)
                moved++;
        }
        return moved;
    }

    // Consistent hashing: only keys between old and new server positions move
    public static int ConsistentHashKeysRemapped(List<string> keys, int oldShards, int newShards)
    {
        var ringBefore = BuildRing(oldShards);
        var ringAfter  = BuildRing(newShards);

        int moved = 0;
        foreach (var key in keys)
        {
            if (GetServer(key, ringBefore) != GetServer(key, ringAfter))
                moved++;
        }
        return moved;
    }

    private static (Dictionary<uint, int> ring, List<uint> keys) BuildRing(int shardCount)
    {
        var ring = new Dictionary<uint, int>();
        var keys = new List<uint>();

        for (int i = 0; i < shardCount; i++)
        {
            uint pos = StableHash($"shard:{i}");
            ring[pos] = i;
            int idx = keys.BinarySearch(pos);
            if (idx < 0) keys.Insert(~idx, pos);
        }

        return (ring, keys);
    }

    private static int GetServer(string key, (Dictionary<uint, int> ring, List<uint> keys) r)
    {
        uint pos = StableHash(key);
        int idx = r.keys.BinarySearch(pos);
        if (idx < 0) idx = ~idx;
        if (idx >= r.keys.Count) idx = 0;
        return r.ring[r.keys[idx]];
    }

    private static uint StableHash(string key)
    {
        using var md5 = MD5.Create();
        byte[] bytes = md5.ComputeHash(Encoding.UTF8.GetBytes(key));
        return BitConverter.ToUInt32(bytes, 0);
    }
}

// Usage
var keys = Enumerable.Range(0, 10000).Select(i => $"key:{i}").ToList();

int moduloMoved    = ReshardingCalculator.ModuloKeysRemapped(keys, 4, 5);
int consistentMoved = ReshardingCalculator.ConsistentHashKeysRemapped(keys, 4, 5);

Console.WriteLine($"Modulo hashing:      {moduloMoved} / {keys.Count} keys moved ({moduloMoved * 100.0 / keys.Count:F1}%)");
Console.WriteLine($"Consistent hashing:  {consistentMoved} / {keys.Count} keys moved ({consistentMoved * 100.0 / keys.Count:F1}%)");

// Expected output:
// Modulo hashing:     ~8000 / 10000 keys moved (~80%)
// Consistent hashing: ~2000 / 10000 keys moved (~20%)  ← 1/N = 1/5
```

### What to Say in the Interview
```
Going from 4 to 5 shards:

Modulo hashing:
  key:5 → 5%4=1, 5%5=0 → MOVES
  key:6 → 6%4=2, 6%5=1 → MOVES
  → Nearly all keys move — this causes massive cache invalidation
    and thundering herd on the new shard

Consistent hashing:
  → Only keys between shard_4's position and the next server move
  → That's 1/N = 20% of keys
  → Everything else stays put
```

## Q4. Hotspot Detector

> Given a stream of key accesses, detect which shard is receiving disproportionately more traffic. A shard is a hotspot if it receives more than `threshold` times its fair share.

```csharp
public class HotspotDetector
{
    private readonly int _shardCount;
    private readonly double _threshold;
    private readonly Dictionary<int, int> _shardHits = new();
    private int _totalRequests = 0;

    public HotspotDetector(int shardCount, double threshold = 2.0)
    {
        _shardCount = shardCount;
        _threshold = threshold;
        for (int i = 0; i < shardCount; i++)
            _shardHits[i] = 0;
    }

    public void RecordRequest(string key)
    {
        int shard = Math.Abs(key.GetHashCode()) % _shardCount;
        _shardHits[shard]++;
        _totalRequests++;
    }

    public List<int> GetHotspots()
    {
        if (_totalRequests == 0) return new List<int>();

        double fairShare = (double)_totalRequests / _shardCount;
        var hotspots = new List<int>();

        foreach (var (shard, hits) in _shardHits)
        {
            if (hits > fairShare * _threshold)
                hotspots.Add(shard);
        }

        return hotspots;
    }

    public void PrintReport()
    {
        double fairShare = _totalRequests > 0 ? (double)_totalRequests / _shardCount : 0;
        Console.WriteLine($"\nTotal requests: {_totalRequests}  |  Fair share per shard: {fairShare:F0}");
        Console.WriteLine($"Hotspot threshold: {_threshold}x fair share = {fairShare * _threshold:F0} hits\n");

        foreach (var (shard, hits) in _shardHits)
        {
            double ratio = fairShare > 0 ? hits / fairShare : 0;
            string flag = hits > fairShare * _threshold ? " ← HOTSPOT" : "";
            Console.WriteLine($"Shard {shard}: {hits,5} hits ({ratio:F2}x fair share){flag}");
        }
    }
}

// Usage — simulate a celebrity user causing hotspot
var detector = new HotspotDetector(shardCount: 4, threshold: 2.0);

// Normal traffic: distributed across many users
var rand = new Random(42);
for (int i = 0; i < 1000; i++)
    detector.RecordRequest($"user:{rand.Next(1000)}");

// Celebrity spike: one user getting 500 extra hits
for (int i = 0; i < 500; i++)
    detector.RecordRequest("user:celebrity_99");

detector.PrintReport();
var hotspots = detector.GetHotspots();
Console.WriteLine($"\nHotspot shards: [{string.Join(", ", hotspots)}]");

// Example output:
// Total requests: 1500  |  Fair share per shard: 375
// Hotspot threshold: 2x fair share = 750 hits
//
// Shard 0:   312 hits (0.83x fair share)
// Shard 1:   847 hits (2.26x fair share) ← HOTSPOT
// Shard 2:   198 hits (0.53x fair share)
// Shard 3:   143 hits (0.38x fair share)
//
// Hotspot shards: [1]
```

### How to Fix Hotspots (Say This in the Interview)
```
1. Key salting: append random suffix to the hot key
   "celebrity_99" → "celebrity_99#0", "celebrity_99#1", ..., "celebrity_99#9"
   → Spreads across 10 shards; reads must query all 10 and merge

2. Dedicated shard: route the celebrity user to their own shard

3. Caching layer: put Redis in front — cache reads for hot keys
   → Shard only sees 1 write per TTL instead of N reads

4. Read replicas: hot shard gets extra replicas to absorb read traffic
```

## Complexity Summary

| Operation | Hash Sharding | Range Sharding | Consistent Hashing |
|-----------|---------------|----------------|--------------------|
| GetShard | O(1) | O(S) | O(log N) |
| AddShard (keys moved) | ~80% | varies | ~1/N |
| Hotspot risk | High (skewed keys) | High (skewed ranges) | Low (virtual nodes) |

S = number of shards, N = total ring positions

---

## What Interviewers Test

```
1. Do you know WHY modulo hashing causes massive resharding?
   → shard = hash(key) % N changes for most keys when N changes
   → Quantify: ~80% of keys move vs ~20% with consistent hashing

2. Can you identify range-based hotspots?
   → Popular prefixes (A, B, C) overload early shards
   → Fix: rebalance ranges, or switch to hash-based routing

3. Do you detect hotspots correctly?
   → Compare shard hits against fair share, not absolute count
   → Fair share = totalRequests / shardCount

4. Do you know the real-world fixes for hotspots?
   → Key salting, caching, dedicated shards, read replicas

5. When would you choose each sharding strategy?
   → Hash: uniform distribution, no range queries needed
   → Range: range scans (dates, IDs) — accept hotspot risk
   → Consistent: distributed caches, need minimal resharding
```
