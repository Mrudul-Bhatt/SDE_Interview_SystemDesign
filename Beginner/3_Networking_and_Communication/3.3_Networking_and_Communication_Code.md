## Q1. Implement a Token Bucket Rate Limiter

> Allow up to `N` requests per second per client. Requests beyond the limit are rejected. The bucket refills at a steady rate, allowing short bursts up to the bucket capacity.

### How Token Bucket Works
```
Bucket capacity = 10 tokens
Refill rate     = 10 tokens/second (1 token every 100ms)

Client sends 10 requests instantly → all allowed (drains bucket)
Client sends 11th request         → rejected (bucket empty)
After 500ms, bucket has 5 tokens  → 5 more requests allowed
```

```csharp
using System;
using System.Collections.Generic;

public class TokenBucketRateLimiter
{
    private class Bucket
    {
        public double Tokens     { get; set; }
        public DateTime LastRefill { get; set; }
    }

    private readonly double _capacity;      // max tokens in bucket
    private readonly double _refillRate;    // tokens added per second
    private readonly Dictionary<string, Bucket> _buckets = new();
    private readonly object _lock = new();

    public TokenBucketRateLimiter(double capacity, double refillPerSecond)
    {
        _capacity   = capacity;
        _refillRate = refillPerSecond;
    }

    public bool Allow(string clientId)
    {
        lock (_lock)
        {
            if (!_buckets.TryGetValue(clientId, out var bucket))
            {
                bucket = new Bucket { Tokens = _capacity, LastRefill = DateTime.UtcNow };
                _buckets[clientId] = bucket;
            }

            // Refill tokens based on elapsed time
            double elapsed = (DateTime.UtcNow - bucket.LastRefill).TotalSeconds;
            bucket.Tokens    = Math.Min(_capacity, bucket.Tokens + elapsed * _refillRate);
            bucket.LastRefill = DateTime.UtcNow;

            if (bucket.Tokens >= 1.0)
            {
                bucket.Tokens--;
                return true;  // request allowed
            }

            return false; // rate limited
        }
    }
}

// Usage — 5 requests/second, burst up to 5
var limiter = new TokenBucketRateLimiter(capacity: 5, refillPerSecond: 5);

for (int i = 0; i < 7; i++)
{
    bool allowed = limiter.Allow("user:alice");
    Console.WriteLine($"Request {i + 1}: {(allowed ? "ALLOWED" : "DENIED")}");
}
// Request 1: ALLOWED
// Request 2: ALLOWED
// Request 3: ALLOWED
// Request 4: ALLOWED
// Request 5: ALLOWED
// Request 6: DENIED   ← bucket empty
// Request 7: DENIED

System.Threading.Thread.Sleep(1000); // wait 1 second → bucket refills
Console.WriteLine($"After 1s: {(limiter.Allow("user:alice") ? "ALLOWED" : "DENIED")}");
// After 1s: ALLOWED
```

## Q2. Implement a Sliding Window Rate Limiter

> Stricter than token bucket — count requests in the last N seconds exactly. Reject if the count exceeds the limit. No burst allowance beyond the window limit.

### Token Bucket vs Sliding Window
```
Token Bucket (capacity=5, rate=5/s):
  t=0.0s: 5 requests → all ALLOWED (used burst)
  t=0.9s: 4 requests → all ALLOWED (bucket has ~4.5 tokens)
  → 9 requests in 1 second window — burst allowed

Sliding Window (limit=5 per second):
  t=0.0s: 5 requests → all ALLOWED
  t=0.1s: 1 request  → DENIED (6 requests in last 1s)
  → Strictly ≤5 requests in any 1-second window
```

```csharp
public class SlidingWindowRateLimiter
{
    private readonly int _limit;
    private readonly TimeSpan _window;
    // clientId → sorted list of request timestamps
    private readonly Dictionary<string, Queue<DateTime>> _timestamps = new();
    private readonly object _lock = new();

    public SlidingWindowRateLimiter(int requestLimit, TimeSpan window)
    {
        _limit  = requestLimit;
        _window = window;
    }

    public bool Allow(string clientId)
    {
        lock (_lock)
        {
            DateTime now = DateTime.UtcNow;
            DateTime windowStart = now - _window;

            if (!_timestamps.TryGetValue(clientId, out var timestamps))
            {
                timestamps = new Queue<DateTime>();
                _timestamps[clientId] = timestamps;
            }

            // Remove timestamps outside the current window
            while (timestamps.Count > 0 && timestamps.Peek() < windowStart)
                timestamps.Dequeue();

            if (timestamps.Count < _limit)
            {
                timestamps.Enqueue(now);
                return true;  // allowed
            }

            return false; // rate limited
        }
    }

    public int GetRequestCount(string clientId)
    {
        lock (_lock)
        {
            if (!_timestamps.TryGetValue(clientId, out var timestamps)) return 0;
            DateTime windowStart = DateTime.UtcNow - _window;
            while (timestamps.Count > 0 && timestamps.Peek() < windowStart)
                timestamps.Dequeue();
            return timestamps.Count;
        }
    }
}

// Usage — 3 requests per 2 seconds
var limiter = new SlidingWindowRateLimiter(requestLimit: 3, window: TimeSpan.FromSeconds(2));

Console.WriteLine(limiter.Allow("user:bob")); // true  (count: 1)
Console.WriteLine(limiter.Allow("user:bob")); // true  (count: 2)
Console.WriteLine(limiter.Allow("user:bob")); // true  (count: 3)
Console.WriteLine(limiter.Allow("user:bob")); // false (count: 3, at limit)

System.Threading.Thread.Sleep(2100); // window slides past all 3 timestamps

Console.WriteLine(limiter.Allow("user:bob")); // true  (window is clear)
```

### When to Use Each
```
Token Bucket:
  → API gateway where clients can burst briefly
  → Allows bursting up to capacity, then throttles
  → Used by: AWS API Gateway, Stripe API

Sliding Window:
  → Stricter fairness — no burst beyond the limit
  → Simpler to reason about: "max N in last T seconds"
  → Used by: GitHub API (5000 req/hr), Redis rate limiting recipes
```

**Complexity:** Allow O(n) worst case (pruning old timestamps), O(1) amortized — each timestamp added once and removed once

## Q3. Implement a Reliable UDP Simulator

> Simulate TCP-like reliability over UDP: assign sequence numbers to packets, send ACKs, and retransmit lost packets. This demonstrates understanding of how TCP guarantees delivery that UDP does not.

```csharp
public class ReliableUDPSender
{
    private int _nextSeqNum = 0;
    private readonly Dictionary<int, string> _unacknowledged = new();
    private readonly int _timeoutMs;
    private readonly Dictionary<int, DateTime> _sentAt = new();

    // Simulate packet loss — drop every Nth packet
    private readonly int _dropEveryN;
    private int _sendCount = 0;

    public ReliableUDPSender(int timeoutMs = 500, int dropEveryN = 3)
    {
        _timeoutMs  = timeoutMs;
        _dropEveryN = dropEveryN;
    }

    public void Send(string data)
    {
        int seqNum = _nextSeqNum++;
        _unacknowledged[seqNum] = data;
        _sentAt[seqNum] = DateTime.UtcNow;
        TransmitPacket(seqNum, data, isRetransmit: false);
    }

    private void TransmitPacket(int seqNum, string data, bool isRetransmit)
    {
        _sendCount++;
        bool dropped = (_sendCount % _dropEveryN == 0);

        if (dropped)
        {
            Console.WriteLine($"  [DROPPED] Seq={seqNum} data='{data}'" +
                              (isRetransmit ? " (retransmit)" : ""));
        }
        else
        {
            Console.WriteLine($"  [SENT]    Seq={seqNum} data='{data}'" +
                              (isRetransmit ? " (retransmit)" : ""));
            // Simulate receiver processing and sending ACK back
            ReceiveAck(seqNum);
        }
    }

    public void ReceiveAck(int seqNum)
    {
        if (_unacknowledged.ContainsKey(seqNum))
        {
            Console.WriteLine($"  [ACK]     Seq={seqNum} acknowledged");
            _unacknowledged.Remove(seqNum);
            _sentAt.Remove(seqNum);
        }
    }

    // Check for timed-out packets and retransmit
    public void RetransmitTimedOut()
    {
        var now = DateTime.UtcNow;
        foreach (var (seqNum, sentAt) in new Dictionary<int, DateTime>(_sentAt))
        {
            if ((now - sentAt).TotalMilliseconds > _timeoutMs)
            {
                Console.WriteLine($"  [TIMEOUT] Seq={seqNum} — retransmitting");
                _sentAt[seqNum] = now;
                TransmitPacket(seqNum, _unacknowledged[seqNum], isRetransmit: true);
            }
        }
    }

    public int UnacknowledgedCount => _unacknowledged.Count;
}

// Usage
var sender = new ReliableUDPSender(timeoutMs: 200, dropEveryN: 3);

Console.WriteLine("=== Sending packets (every 3rd is dropped) ===");
sender.Send("Hello");    // seq=0
sender.Send("World");    // seq=1
sender.Send("Dropped");  // seq=2  ← simulated drop

Console.WriteLine($"\nUnacknowledged: {sender.UnacknowledgedCount}");

System.Threading.Thread.Sleep(250); // wait for timeout

Console.WriteLine("\n=== Checking for timeouts and retransmitting ===");
sender.RetransmitTimedOut(); // seq=2 gets retransmitted

Console.WriteLine($"Unacknowledged after retry: {sender.UnacknowledgedCount}");
```

### What This Demonstrates
```
TCP's reliability over UDP is achieved by:

1. Sequence numbers  — receiver knows if a packet is missing or out-of-order
2. ACKs             — sender knows which packets arrived
3. Retransmission   — sender re-sends after timeout with no ACK received
4. Ordering         — receiver buffers out-of-order packets, delivers in order

UDP has none of this — you get it in applications like:
  - QUIC (HTTP/3): implements reliability selectively, per stream
  - Game engines:  reliable for events (player join), unreliable for position
```

## Q4. Implement a Path-Based Request Router

> A reverse proxy routes incoming requests to different backend services based on URL path prefix. Implement a router that supports prefix matching, wildcard routes, and middleware (auth, logging).

```csharp
public class RequestRouter
{
    public record HttpRequest(string Method, string Path, Dictionary<string, string> Headers);
    public record HttpResponse(int StatusCode, string Body);

    private class Route
    {
        public string Prefix    { get; init; } = "";
        public string Backend   { get; init; } = "";
        public bool RequireAuth { get; init; }
    }

    private readonly List<Route> _routes = new();
    private readonly HashSet<string> _validTokens = new();

    public void AddRoute(string prefix, string backend, bool requireAuth = false)
    {
        _routes.Add(new Route { Prefix = prefix, Backend = backend, RequireAuth = requireAuth });
        // Sort by prefix length descending — more specific routes take priority
        _routes.Sort((a, b) => b.Prefix.Length.CompareTo(a.Prefix.Length));
    }

    public void AddValidToken(string token) => _validTokens.Add(token);

    public HttpResponse Handle(HttpRequest request)
    {
        // Logging middleware
        Console.WriteLine($"[LOG] {request.Method} {request.Path}");

        // Find matching route (longest prefix wins)
        Route? matched = null;
        foreach (var route in _routes)
        {
            if (request.Path.StartsWith(route.Prefix))
            {
                matched = route;
                break;
            }
        }

        if (matched == null)
            return new HttpResponse(404, "No route found");

        // Auth middleware
        if (matched.RequireAuth)
        {
            request.Headers.TryGetValue("Authorization", out string? token);
            if (token == null || !_validTokens.Contains(token))
            {
                Console.WriteLine($"[AUTH] Rejected — missing or invalid token");
                return new HttpResponse(401, "Unauthorized");
            }
        }

        // Forward to backend (simulated)
        Console.WriteLine($"[PROXY] Forwarding to backend: {matched.Backend}");
        return new HttpResponse(200, $"Response from {matched.Backend}");
    }
}

// Usage — simulates an API gateway routing table
var router = new RequestRouter();
router.AddRoute("/api/users",   backend: "user-service:8080",   requireAuth: true);
router.AddRoute("/api/orders",  backend: "order-service:8081",  requireAuth: true);
router.AddRoute("/api/",        backend: "api-service:8082",    requireAuth: false);
router.AddRoute("/static/",     backend: "cdn-origin:9000",     requireAuth: false);
router.AddRoute("/",            backend: "web-server:3000",     requireAuth: false);
router.AddValidToken("Bearer valid-token-123");

var req1 = new RequestRouter.HttpRequest("GET", "/api/users/1",
    new Dictionary<string, string> { ["Authorization"] = "Bearer valid-token-123" });
var res1 = router.Handle(req1);
Console.WriteLine($"→ {res1.StatusCode}: {res1.Body}\n");

var req2 = new RequestRouter.HttpRequest("GET", "/api/users/1",
    new Dictionary<string, string>()); // no auth token
var res2 = router.Handle(req2);
Console.WriteLine($"→ {res2.StatusCode}: {res2.Body}\n");

var req3 = new RequestRouter.HttpRequest("GET", "/static/logo.png",
    new Dictionary<string, string>());
var res3 = router.Handle(req3);
Console.WriteLine($"→ {res3.StatusCode}: {res3.Body}\n");

var req4 = new RequestRouter.HttpRequest("GET", "/unknown/path",
    new Dictionary<string, string>());
var res4 = router.Handle(req4);
Console.WriteLine($"→ {res4.StatusCode}: {res4.Body}");

// Output:
// [LOG] GET /api/users/1
// [PROXY] Forwarding to backend: user-service:8080
// → 200: Response from user-service:8080

// [LOG] GET /api/users/1
// [AUTH] Rejected — missing or invalid token
// → 401: Unauthorized

// [LOG] GET /static/logo.png
// [PROXY] Forwarding to backend: cdn-origin:9000
// → 200: Response from cdn-origin:9000

// [LOG] GET /unknown/path
// → 404: No route found
```

### Longest Prefix Match — Why Order Matters
```
Routes sorted by prefix length (most specific first):
  /api/users   → user-service    (length 10)
  /api/orders  → order-service   (length 11)
  /api/        → api-service     (length 5)
  /static/     → cdn-origin      (length 8)
  /            → web-server      (length 1)

Request: GET /api/users/profile
  Matches /api/users  ✓  (stops here — longest match wins)
  Would also match /api/ and / — but those are less specific
```

## Q5. Implement an In-Memory Chat Room (WebSocket Message Broadcaster)

> Multiple clients connect to a chat room. When one client sends a message, all other connected clients receive it. Handle clients joining and leaving gracefully.

```csharp
public class ChatRoom
{
    private readonly string _roomId;
    // In real WebSocket code this would be WebSocket connections;
    // here we simulate with client IDs and message queues
    private readonly Dictionary<string, Queue<string>> _clients = new();
    private readonly List<string> _history = new();
    private readonly int _historyLimit;
    private readonly object _lock = new();

    public ChatRoom(string roomId, int historyLimit = 50)
    {
        _roomId       = roomId;
        _historyLimit = historyLimit;
    }

    public void Join(string clientId)
    {
        lock (_lock)
        {
            _clients[clientId] = new Queue<string>();
            Console.WriteLine($"[{_roomId}] {clientId} joined ({_clients.Count} online)");

            // Send recent history to new client
            foreach (var msg in _history)
                _clients[clientId].Enqueue(msg);
        }
    }

    public void Leave(string clientId)
    {
        lock (_lock)
        {
            _clients.Remove(clientId);
            Console.WriteLine($"[{_roomId}] {clientId} left ({_clients.Count} online)");
            Broadcast("system", $"{clientId} has left the room");
        }
    }

    public void SendMessage(string senderId, string text)
    {
        lock (_lock)
        {
            if (!_clients.ContainsKey(senderId))
            {
                Console.WriteLine($"[ERROR] {senderId} is not in room {_roomId}");
                return;
            }

            string message = $"[{DateTime.UtcNow:HH:mm:ss}] {senderId}: {text}";
            _history.Add(message);
            if (_history.Count > _historyLimit)
                _history.RemoveAt(0);

            // Broadcast to all OTHER clients (not the sender)
            foreach (var (clientId, inbox) in _clients)
                if (clientId != senderId)
                    inbox.Enqueue(message);

            Console.WriteLine($"[SENT] {message}");
        }
    }

    private void Broadcast(string sender, string text)
    {
        string message = $"[{DateTime.UtcNow:HH:mm:ss}] {sender}: {text}";
        foreach (var (_, inbox) in _clients)
            inbox.Enqueue(message);
    }

    // Simulate client reading its messages (in real code: WebSocket.SendAsync)
    public List<string> ReadMessages(string clientId)
    {
        lock (_lock)
        {
            var messages = new List<string>();
            if (!_clients.TryGetValue(clientId, out var inbox)) return messages;
            while (inbox.Count > 0) messages.Add(inbox.Dequeue());
            return messages;
        }
    }
}

// Usage
var room = new ChatRoom("general");

room.Join("alice");
room.Join("bob");
room.Join("charlie");

room.SendMessage("alice", "Hey everyone!");
room.SendMessage("bob",   "Hi Alice!");

// Bob reads his messages
var bobMessages = room.ReadMessages("bob");
Console.WriteLine("\nBob's inbox:");
foreach (var msg in bobMessages) Console.WriteLine($"  {msg}");

room.Leave("charlie");

var aliceMessages = room.ReadMessages("alice");
Console.WriteLine("\nAlice's inbox:");
foreach (var msg in aliceMessages) Console.WriteLine($"  {msg}");

// Output:
// [general] alice joined (1 online)
// [general] bob joined (2 online)
// [general] charlie joined (3 online)
// [SENT] [12:00:00] alice: Hey everyone!
// [SENT] [12:00:00] bob: Hi Alice!
// [general] charlie left (2 online)
//
// Bob's inbox:
//   [12:00:00] alice: Hey everyone!
//
// Alice's inbox:
//   [12:00:00] bob: Hi Alice!
//   [12:00:00] system: charlie has left the room
```

### Scaling Chat Beyond One Server
```
Single server (this implementation):
  → All clients on same machine → simple in-memory broadcast
  → Works up to ~10k concurrent connections per server

Multi-server (real-world):
  → Alice on Server 1, Bob on Server 2
  → Alice's message arrives at Server 1
  → Server 1 can't reach Bob directly

Solution — Pub/Sub with Redis:
  Server 1: receives message → publishes to Redis channel "room:general"
  Server 2: subscribed to "room:general" → receives message → pushes to Bob

  [Alice] → [Server 1] → Redis Pub/Sub → [Server 2] → [Bob]
                                       → [Server 3] → [Charlie]
```

## Complexity Summary

| Algorithm | Allow/Handle | Space | Best For |
|-----------|-------------|-------|----------|
| Token Bucket | O(1) | O(clients) | Burst-tolerant rate limiting |
| Sliding Window | O(1) amortized | O(clients × window) | Strict per-window limits |
| Reliable UDP | O(unacked) | O(unacked packets) | Protocol design questions |
| Path Router | O(routes) | O(routes) | API gateway / reverse proxy |
| Chat Room | O(clients) | O(clients × history) | Real-time messaging |

## What Interviewers Test

```
1. Rate Limiter — which algorithm did you pick and why?
   → Token Bucket: allows bursting, good for APIs
   → Sliding Window: no burst, strict fairness
   → Fixed Window: simplest but allows 2x burst at window boundary (bug!)

2. Token Bucket — is the refill calculation correct?
   → tokens = min(capacity, tokens + elapsed * rate)
   → Must cap at capacity — forgetting min() gives infinite tokens

3. Reliable UDP — do you know the three mechanisms?
   → Sequence numbers + ACKs + Retransmission on timeout
   → Bonus: mention QUIC does this per-stream (no head-of-line blocking)

4. Path Router — do you handle prefix priority?
   → /api/users must win over /api/ — sort by length descending
   → Forgetting this sends all /api/users traffic to wrong backend

5. Chat Room scaling — what breaks at multiple servers?
   → In-memory broadcast doesn't cross server boundaries
   → Answer: Redis Pub/Sub, or Kafka topic per room
```
